# IDX-004 — Reorg Handling

> **Authoritative requirement:** [IDX-004](../NORMATIVE.md#IDX-004)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **CHIP reference:** [chip-groth16-l2-consensus.md](../../../../resources/chip-groth16-l2-consensus.md) — Part 4: Off-Chain Validator Set Construction

## Summary

On blockchain reorganization, the indexer rolls back to the last safe checkpoint before the reorg point and re-indexes forward. If no safe point exists, a full re-index from genesis is required.

## Specification

### Reorg Detection

A reorg is detected when the node reports a peak height lower than the indexer's last synced height:

```rust
if peak < self.last_synced_height {
    self.handle_reorg(node, config, peak).await?;
}
```

### Rollback Algorithm

1. Find the last checkpoint confirmed at or before the new peak height
2. If found: retain checkpoint history up to that epoch, clear registration coins, re-sync from that height
3. If not found: perform full re-index from network deployment

### Full Re-index

When no safe checkpoint exists:

```rust
async fn full_reindex(&mut self, node: &FullNodeClient, config: &NetworkConfig) {
    self.registration_coins.clear();
    self.checkpoint_history.clear();
    self.last_synced_height = 0;

    // Start from network coin launcher height
    let launcher_height = node
        .get_coin_record_by_name(config.network_coin_launcher_id)
        .await?
        .confirmed_block_index;

    self.last_synced_height = launcher_height.saturating_sub(1);
    self.sync(node, config).await
}
```

### Why This Design

- Checkpoint history provides recovery points
- Rolling back to a known-good state is faster than full re-index
- Full re-index is the ultimate fallback for safety

## Acceptance Criteria

- [ ] Reorg detected when peak < last_synced_height
- [ ] Rollback finds last safe checkpoint before reorg point
- [ ] Registration coins are cleared before re-sync
- [ ] Full re-index triggered when no safe point exists
- [ ] State is consistent after reorg handling

## Implementation Notes

- **Module:** `src/indexer/reorg.rs`
- **Function:** `handle_reorg()`
- **Dependencies:** Checkpoint history

### Reorg Handler

```rust
async fn handle_reorg(
    &mut self,
    node: &FullNodeClient,
    config: &NetworkConfig,
    new_peak: u32,
) -> Result<(), IndexerError> {
    let safe_checkpoint = self.checkpoint_history
        .iter()
        .rev()
        .find(|c| c.confirmed_at_height <= new_peak)
        .cloned();

    if let Some(safe) = safe_checkpoint {
        self.checkpoint_history.retain(|c| c.epoch <= safe.epoch);
        self.registration_coins.clear();
        self.last_synced_height = safe.confirmed_at_height;
        self.sync(node, config).await?;
    } else {
        self.full_reindex(node, config).await?;
    }

    Ok(())
}
```
