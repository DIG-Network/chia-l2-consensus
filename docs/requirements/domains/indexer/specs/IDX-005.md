# IDX-005 — Persistent Cache

> **Authoritative requirement:** [IDX-005](../NORMATIVE.md#IDX-005)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **CHIP reference:** [chip-groth16-l2-consensus.md](../../../../resources/chip-groth16-l2-consensus.md) — Part 4: Off-Chain Validator Set Construction

## Summary

The indexer maintains a persistent cache (JSON file) using atomic writes to enable fast restarts without full re-indexing. This is a SHOULD requirement as it improves performance but is not strictly required for correctness.

## Specification

### Cache Contents

```rust
#[derive(Serialize, Deserialize)]
pub struct IndexerCache {
    pub last_synced_height: u32,
    pub checkpoint_history: Vec<CheckpointRecord>,
    pub registration_coins: Vec<RegistrationCoinRecord>,
    pub network_coin_state: SerializableNetworkCoinState,
    pub checkpoint_state: SerializableCheckpointState,
}
```

### Atomic Writes

Cache updates use atomic writes to prevent corruption:

```rust
pub fn save(&self, path: &str) -> Result<(), CacheError> {
    let bytes = serde_json::to_vec_pretty(self)?;
    let tmp = format!("{}.tmp", path);
    std::fs::write(&tmp, &bytes)?;
    std::fs::rename(&tmp, path)?;  // atomic write
    Ok(())
}
```

### Cache Invalidation

The cache must be deleted and rebuilt when:
- Network is redeployed (new launcher IDs)
- Cache file is corrupted
- Merkle consistency check fails

### Cache Path Convention

The cache path should include launcher IDs to prevent stale cache issues across deployments:

```
indexer_cache_{network_coin_launcher_id}_{checkpoint_launcher_id}.json
```

## Acceptance Criteria

- [ ] Cache persists indexer state to disk
- [ ] Atomic writes prevent partial cache corruption
- [ ] Fast restart from cached state
- [ ] Cache invalidation on redeployment
- [ ] JSON format for easy inspection

## Implementation Notes

- **Module:** `src/indexer/cache.rs`
- **Format:** JSON (human-readable)
- **Write strategy:** Atomic rename

### Load Function

```rust
impl IndexerCache {
    pub fn load(path: &str) -> Result<Option<IndexerCache>, CacheError> {
        if !std::path::Path::new(path).exists() {
            return Ok(None);
        }
        let bytes = std::fs::read(path)?;
        Ok(Some(serde_json::from_slice(&bytes)?))
    }
}
```

## Verification

1. Sync indexer and verify cache created
2. Stop indexer and restart
3. Verify fast resume from cache (no full re-index)
4. Corrupt cache file
5. Verify graceful recovery (full re-index)
