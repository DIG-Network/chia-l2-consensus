# IDX-002 — Lineage Verification

> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

The indexer verifies each registration coin's lineage by confirming its parent coin ID is in the set of valid network coin spend IDs. This is the core security mechanism that prevents fake registrations. Coins failing this check are ignored.

## Specification

### Lineage Check Algorithm

1. **Check parent validity**: Registration coin's `parent_coin_info` must be a known network coin spend ID
2. **Extract pubkey**: Parse the validator pubkey from the parent spend memo
3. **Verify puzzle hash**: Computed puzzle hash must match the coin's actual puzzle hash
4. **Verify collateral**: Coin amount must equal the required collateral amount

### Why Lineage Matters

Without lineage verification:
- Anyone could create a coin with the registration puzzle hash
- Fake validators could be added to the validator set
- The majority assumption would be undermined

Because coin IDs are deterministic and the blockchain is immutable, this check cannot be gamed.

### Pubkey Extraction from Memo

The validator pubkey is stored as a memo on the registration coin creation condition:
- First memo on the CreateCoin condition matching the child coin
- Must be exactly 48 bytes (G1 compressed)
- Parsed as BLS12-381 public key

### Puzzle Hash Verification

```rust
let expected_hash = registration_coin_puzzle_hash(
    config.registration_coin_mod_hash,
    pubkey,
    config.checkpoint_singleton_id(),
);
if coin.puzzle_hash != expected_hash {
    return Err(IndexerError::PuzzleHashMismatch);
}
```

## Acceptance Criteria

- [ ] Only coins with valid network coin parent are indexed
- [ ] Pubkey is correctly extracted from memo
- [ ] Puzzle hash is verified against computed expected hash
- [ ] Collateral amount is verified
- [ ] Invalid coins are silently ignored (not errored)

## Implementation Notes

- **Module:** `src/indexer/validator_set.rs`
- **Function:** `try_parse_registration_coin()`
- **Dependencies:** Network coin spend tracking

### Detection Logic

```rust
async fn try_parse_registration_coin(
    &self,
    node: &FullNodeClient,
    coin: &Coin,
    config: &NetworkConfig,
    height: u32,
) -> Result<Option<RegistrationCoinRecord>, IndexerError> {
    // Step 1: Check if parent is a known network coin spend
    let is_valid_parent = self.network_coin_spend_ids.contains(
        &coin.parent_coin_info
    );
    if !is_valid_parent {
        return Ok(None);
    }

    // Steps 2-4: Extract pubkey, verify puzzle hash, verify collateral
    // ...
}
```

## Verification

1. Register validator through proper network coin spend
2. Verify indexer picks up the registration coin
3. Create coin with registration puzzle hash but wrong parent
4. Verify indexer ignores it
5. Verify validator set only includes legitimate registrations
