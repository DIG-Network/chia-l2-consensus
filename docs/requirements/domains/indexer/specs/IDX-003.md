# IDX-003 — Merkle Consistency

> **Authoritative requirement:** [IDX-003](../NORMATIVE.md#IDX-003)

> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

After every sync, the indexer rebuilds the sparse Merkle tree from registration coins and verifies the computed root matches the on-chain `validator_merkle_root`. Mismatches return `StateMismatch` error and trigger investigation.

## Specification

### Consistency Check Algorithm

1. Build sparse Merkle tree from indexed registration coins
2. Compute the tree root
3. Compare against checkpoint singleton's `validator_merkle_root`
4. If mismatch, return `MerkleRootMismatch` error

### What Mismatch Means

A `MerkleRootMismatch` error indicates one of:
- The indexer missed some registration coin additions or removals
- Bug in the sparse Merkle tree implementation
- Corrupted indexer cache

In any case, a full re-index is triggered.

### Tree Construction

The tree is built using the algorithm from the sparse Merkle tree specification:
- Validators are sorted by pubkey bytes for deterministic ordering
- Each pubkey is hashed to determine its slot
- Tree depth matches the configured `TREE_DEPTH`

## Acceptance Criteria

- [ ] Merkle tree is rebuilt after every sync
- [ ] Computed root is compared to on-chain root
- [ ] Mismatch returns `StateMismatch` error
- [ ] Full re-index is triggered on mismatch
- [ ] Match confirms local state is consistent

## Implementation Notes

- **Module:** `src/indexer/mod.rs`
- **Function:** `verify_merkle_consistency()`
- **Dependencies:** Sparse Merkle tree implementation

### Verification Function

```rust
pub fn verify_merkle_consistency(
    validator_set: &ValidatorSet,
    checkpoint: &CheckpointSingletonState,
    tree_depth: u32,
) -> Result<SparseMerkleTree, IndexerError> {
    let tree = SparseMerkleTree::from_validators(
        &validator_set.validators,
        tree_depth,
    );

    let computed_root = tree.root();

    if computed_root != checkpoint.validator_merkle_root {
        return Err(IndexerError::MerkleRootMismatch {
            computed: hex::encode(computed_root),
            on_chain: hex::encode(checkpoint.validator_merkle_root),
        });
    }

    Ok(tree)
}
```

## Verification

1. Sync indexer with clean state
2. Verify Merkle root matches on-chain
3. Corrupt local cache (remove a registration)
4. Run sync and verify mismatch detected
5. Verify full re-index restores consistency
