## Summary

The indexer maintains a reliable local view of on-chain state that the ConsensusClient uses to build spend bundles and generate proofs. It tracks four categories of state: network coin, checkpoint singleton, registration coins, and checkpoint history.

## Specification

### Tracked State Categories

| Category | What is Tracked | Purpose |
|----------|-----------------|---------|
| Network coin state | Current unspent coin, lineage proof, last spend height | Required before every registration |
| Checkpoint singleton state | Current coin, lineage proof, decoded state (epoch, roots, count) | Source of truth for L2 state |
| Registration coins | All unspent registration coins keyed by pubkey | Validator set construction |
| Checkpoint history | Ordered list of all past checkpoint states | Audit and reorg recovery |

### Network Coin State

```rust
pub struct NetworkCoinState {
    pub coin: Coin,
    pub lineage_proof: LineageProof,
    pub last_spend_height: u32,
}
```

### Checkpoint Singleton State

```rust
pub struct CheckpointSingletonState {
    pub coin: Coin,
    pub lineage_proof: LineageProof,
    pub state_root: Bytes32,
    pub epoch: u64,
    pub validator_merkle_root: Bytes32,
    pub validator_count: u64,
}
```

### Registration Coin Records

```rust
pub struct RegistrationCoinRecord {
    pub coin: Coin,
    pub pubkey: PublicKey,
    pub registered_at_height: u32,
    pub registered_at_epoch: u64,
}
```

### Checkpoint History

```rust
pub struct CheckpointRecord {
    pub epoch: u64,
    pub state_root: Bytes32,
    pub validator_merkle_root: Bytes32,
    pub validator_count: u64,
    pub confirmed_at_height: u32,
    pub coin_id: Bytes32,
}
```

## Acceptance Criteria

- [ ] Indexer tracks network coin current state
- [ ] Indexer tracks checkpoint singleton current state
- [ ] Indexer tracks all valid registration coins keyed by pubkey
- [ ] Indexer maintains checkpoint history for reorg recovery
- [ ] State updates atomically on successful sync

## Implementation Notes

- **Module:** `src/indexer/mod.rs`
- **State structure:** `IndexerState`
- **Dependencies:** Full node RPC client

### IndexerState

```rust
pub struct IndexerState {
    pub last_synced_height: u32,
    pub network_coin: NetworkCoinState,
    pub checkpoint: CheckpointSingletonState,
    pub registration_coins: HashMap<PublicKey, RegistrationCoinRecord>,
    pub checkpoint_history: Vec<CheckpointRecord>,
    cache: IndexerCache,
}
```

## Verification

1. Deploy network with all components
2. Run indexer sync
3. Verify network coin state matches on-chain
4. Verify checkpoint singleton state matches on-chain
5. Verify all registration coins are indexed
6. Verify checkpoint history is complete
