# NET-007 — CLVM Execution Validation

> **Authoritative requirement:** [NET-007](../NORMATIVE.md#NET-007)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **CHIP reference:** [chip-groth16-l2-consensus.md](../../../../resources/chip-groth16-l2-consensus.md) — Part 1: Network Coin

## Summary

NET-001 through NET-004 currently have only source-string-inspection tests
(`src.contains()`). Per SCHEMA.md Hard Testing Requirements, puzzle-domain
requirements MUST have CLVM execution tests that actually run the compiled
puzzle. This requirement formalises that gap.

## Specification

For each of NET-001, NET-002, NET-003, NET-004, the test file MUST:

1. **Deserialize** `puzzles/compiled/network_coin_inner.hex` via
   `clvmr::serde::node_from_bytes`
2. **Curry** the puzzle with test parameters using `CurriedProgram`:
   - `REGISTRATION_COIN_MOD_HASH` (32-byte test hash)
   - `COLLATERAL_AMOUNT` (e.g. 1_000_000_000_000)
   - `CHECKPOINT_SINGLETON_ID` (32-byte test ID)
3. **Build** a solution with:
   - `new_validator_pubkey` (48-byte test BLS pubkey)
   - `conditions` (empty list `()`)
4. **Run** via `clvmr::run_program()` with `ChiaDialect`
5. **Parse** the output condition list
6. **Assert** per requirement:

### NET-001 assertions (singleton identity)
- Puzzle compiles and loads into `Allocator`
- Running produces a valid condition list (not an error)
- Condition count is deterministic across runs

### NET-002 assertions (AggSigMe)
- Output contains `AGG_SIG_ME` condition (opcode 50)
- AGG_SIG_ME pubkey == solution pubkey
- AGG_SIG_ME message == `sha256("register" + pubkey)` (cross-impl check)

### NET-003 assertions (registration coin creation)
- Output contains `CREATE_COIN` condition (opcode 51)
- CREATE_COIN puzzle_hash == `curry_tree_hash(MOD_HASH, pubkey, CKPT_ID)`
- CREATE_COIN amount == `COLLATERAL_AMOUNT`
- Different pubkeys produce different puzzle hashes

### NET-004 assertions (self-recreation)
- Verify inner puzzle output does not self-recreate (singleton wrapper handles it)
- OR if inner puzzle emits recreation: verify puzzle_hash and amount=1

## Acceptance Criteria

- [ ] Each of NET-001 through NET-004 has ≥1 CLVM execution test
- [ ] Tests deserialize actual `.hex` artifact (not source string)
- [ ] Tests run the curried puzzle via `run_program()`
- [ ] Tests parse and assert specific output conditions
- [ ] Tests use cross-implementation hash verification where applicable
- [ ] Source-inspection tests remain as supplemental (not sole) verification
