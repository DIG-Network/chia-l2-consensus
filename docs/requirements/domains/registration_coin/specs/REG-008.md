# REG-008 — CLVM Execution Validation

> **Authoritative requirement:** [REG-008](../NORMATIVE.md#REG-008)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

REG-001 has 18 tests but all use source-string inspection (`src.contains()`).
Per SCHEMA.md Hard Testing Requirements, puzzle-domain requirements MUST have
CLVM execution tests. This requirement adds proper CLVM execution coverage.

## Specification

Tests MUST:
1. Deserialize `puzzles/compiled/registration_coin.hex` via `node_from_bytes`
2. Curry with `VALIDATOR_PUBKEY` (48 bytes) and `CHECKPOINT_SINGLETON_ID` (32 bytes)
3. Run with solution: `(epoch, collateral_destination, collateral_amount, ())`
4. Parse output conditions
5. Assert exactly 2 conditions: `ASSERT_COIN_ANNOUNCEMENT` + `CREATE_COIN`
6. Assert `CREATE_COIN` destination and amount match solution
7. Assert `ASSERT_COIN_ANNOUNCEMENT` hash matches the membership announcement spec
8. Test with different pubkeys and checkpoint IDs to verify currying changes output

## Acceptance Criteria

- [x] ≥1 test deserializes `.hex` and runs via `run_program()` — 8 tests
- [x] Output condition count verified — `vv_req_reg_001_clvm_exactly_two_conditions`
- [x] ASSERT_COIN_ANNOUNCEMENT hash verified — `vv_req_reg_001_clvm_announcement_hash_cross_impl` + `_epoch_0`
- [x] CREATE_COIN destination and amount verified — 2 dedicated tests
- [x] Multiple curried parameter combinations tested — different pubkeys and checkpoint IDs
