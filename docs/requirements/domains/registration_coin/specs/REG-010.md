# REG-010 — Simulator Spend Verification

> **Authoritative requirement:** [REG-010](../NORMATIVE.md#REG-010)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

REG-003 through REG-006 have CLVM execution tests (deserialize, curry, run,
parse) but lack simulator-level spend bundle verification. This requirement
adds full consensus-context testing.

## Specification

### REG-003 simulator test (collateral lock)
- Create registration coin in simulator
- Attempt spend WITHOUT checkpoint announcement → rejected
- Verifies lock holds in real consensus

### REG-004 simulator test (announcement assertion)
- Create registration coin + checkpoint singleton
- Build cross-coin bundle with correct announcement
- Verify assertion passes in simulator

### REG-005 simulator test (collateral return)
- Complete the recovery flow in simulator
- Verify destination coin receives full collateral amount
- Verify registration coin is spent

### REG-006 simulator test (epoch replay)
- Register, exit at epoch N, re-register
- Attempt recovery with epoch N announcement at epoch M → rejected
- Verifies epoch replay protection in full consensus

## Acceptance Criteria

- [ ] REG-003: collateral lock verified in simulator
- [ ] REG-004: announcement assertion verified in simulator
- [ ] REG-005: collateral returned to correct destination in simulator
- [ ] REG-006: epoch replay prevented in simulator
