# WDC-001 — Withdraw Delay Coin Puzzle Structure

> **Authoritative requirement:** [WDC-001](../NORMATIVE.md#WDC-001)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [spec-withdraw-delay-coin.md](../../../../resources/spec-withdraw-delay-coin.md)

## Summary

The withdraw delay coin is a simple time-locked puzzle with no solution
parameters and no passthrough conditions. All behavior is fully determined
by the three curried parameters, making the puzzle hash a complete commitment
to destination, amount, and delay.

## Specification

### Curried Parameters

| # | Parameter | Type | Size | Description |
|---|-----------|------|------|-------------|
| 1 | `DESTINATION` | Bytes32 | 32 bytes | Puzzle hash where funds go after delay |
| 2 | `AMOUNT` | Int | variable | Collateral in mojos |
| 3 | `WITHDRAW_DELAY_BLOCKS` | Int | variable | L1 blocks to wait |

### Solution

Empty. No solution parameters are accepted. This prevents:
- Condition injection attacks (SEC-008)
- Destination substitution
- Amount manipulation
- Delay reduction

### Puzzle Output

Exactly two conditions, always:
1. `ASSERT_HEIGHT_RELATIVE(WITHDRAW_DELAY_BLOCKS)`
2. `CREATE_COIN(DESTINATION, AMOUNT)`

### Puzzle Source

```rue
fun main(
    DESTINATION: Bytes32,
    AMOUNT: Int,
    WITHDRAW_DELAY_BLOCKS: Int,
) -> List<Condition> {
    [
        AssertHeightRelative(WITHDRAW_DELAY_BLOCKS),
        CreateCoin(DESTINATION, AMOUNT),
    ]
}
```

### Puzzle Hash Computation

```rust
pub fn withdraw_delay_puzzle_hash(
    mod_hash: Bytes32,
    destination: Bytes32,
    amount: u64,
    delay_blocks: u64,
) -> Bytes32 {
    curry_puzzle_hash(mod_hash, &[
        clvm_encode(&destination),
        clvm_encode(&amount),
        clvm_encode(&delay_blocks),
    ])
}
```

## Acceptance Criteria

- [ ] Puzzle compiles with `rue build`
- [ ] `.hex` and `.hash` artifacts generated
- [ ] Puzzle has exactly 3 curried parameters
- [ ] Solution is empty (nil)
- [ ] Puzzle emits exactly 2 conditions
- [ ] No passthrough conditions accepted from solution
- [ ] Puzzle hash is deterministic from (destination, amount, delay)
- [ ] Different parameters produce different puzzle hashes
