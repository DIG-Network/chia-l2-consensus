# WDC-009 — E2E Simulator Test

> **Authoritative requirement:** [WDC-009](../NORMATIVE.md#WDC-009)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

A full end-to-end simulator test exercises the complete two-phase collateral
recovery lifecycle including the time-locked withdraw delay coin.

## Required Test Scenarios

### Success Path

```
1. Deploy network coin + checkpoint singleton
2. Register validator with collateral
3. Submit checkpoint including validator
4. Submit checkpoint excluding validator (exit)
5. Spend registration coin → creates WITHDRAW DELAY COIN
   - Verify delay coin exists with correct puzzle hash
   - Verify delay coin amount == collateral
6. Advance simulator past WITHDRAW_DELAY_BLOCKS
7. Spend withdraw delay coin → creates coin at DESTINATION
   - Verify destination coin exists
   - Verify destination coin amount == collateral
```

### Failure: Spend Before Delay

```
1. Create withdraw delay coin (steps 1-5 above)
2. Attempt to spend delay coin BEFORE delay period
3. Verify spend is rejected (ASSERT_HEIGHT_RELATIVE failure)
4. Advance to delay_blocks - 1 → still rejected
5. Advance to delay_blocks → now accepted
```

### Failure: Registration Coin Creates Delay Coin (Not Direct)

```
1. After spending registration coin
2. Verify NO coin exists at collateral_destination
3. Verify a coin EXISTS with withdraw_delay_puzzle_hash
```

### Third-Party Release

```
1. Create withdraw delay coin (validator initiates)
2. Advance past delay
3. A DIFFERENT party spends the delay coin
4. Verify funds arrive at validator's DESTINATION
```

## Acceptance Criteria

- [ ] Full lifecycle: deploy → register → checkpoint → exit → delay → release
- [ ] Delay coin created with correct puzzle hash and amount
- [ ] Spend before delay rejected
- [ ] Spend at exact delay boundary accepted
- [ ] Destination coin amount == original collateral
- [ ] No direct destination coin from registration coin spend
- [ ] Third-party release succeeds
- [ ] Default delay = 24,000 blocks used in at least one test

## Implementation Notes

### Simulator Block Advancement

```rust
// chia-sdk-test simulator supports height control:
let creation_height = sim.height();
// ... attempt early spend, expect failure ...
sim.advance_to_height(creation_height + withdraw_delay_blocks);
// ... now spend succeeds ...
```
