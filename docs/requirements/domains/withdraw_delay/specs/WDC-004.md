# WDC-004 — Registration Coin Integration

> **Authoritative requirement:** [WDC-004](../NORMATIVE.md#WDC-004)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [spec-withdraw-delay-coin.md](../../../../resources/spec-withdraw-delay-coin.md)

## Summary

The registration coin puzzle changes from creating a coin directly at the
destination to creating a withdraw delay coin. This requires two new curried
parameters and an on-chain `curry_hash` computation to derive the delay
coin's puzzle hash.

## Specification

### Before (Current)

```rue
// Registration coin output:
CreateCoin(collateral_destination, collateral_amount)
```

### After (With Withdraw Delay)

```rue
// Registration coin computes delay coin puzzle hash on-chain:
let withdraw_delay_puzzle_hash = curry_hash(
    WITHDRAW_DELAY_MOD_HASH,
    collateral_destination,
    collateral_amount,
    WITHDRAW_DELAY_BLOCKS,
);

// Registration coin output:
CreateCoin(withdraw_delay_puzzle_hash, collateral_amount)
```

### New Curried Parameters on Registration Coin

| Parameter | Type | Description |
|-----------|------|-------------|
| `WITHDRAW_DELAY_MOD_HASH` | Bytes32 | Tree hash of uncurried withdraw delay coin puzzle |
| `WITHDRAW_DELAY_BLOCKS` | Int | Block delay, passed through to delay coin curry |

### Updated Registration Coin Puzzle

```rue
fun main(
    VALIDATOR_PUBKEY: PublicKey,
    CHECKPOINT_SINGLETON_ID: Bytes32,
    WITHDRAW_DELAY_MOD_HASH: Bytes32,       // NEW
    WITHDRAW_DELAY_BLOCKS: Int,              // NEW

    epoch: Int,
    collateral_destination: Bytes32,
    collateral_amount: Int,
) -> List<Condition> {

    let withdraw_delay_puzzle_hash = curry_hash(
        WITHDRAW_DELAY_MOD_HASH,
        collateral_destination,
        collateral_amount,
        WITHDRAW_DELAY_BLOCKS,
    );

    let expected_announcement = sha256(
        "membership" + int_to_8_bytes_be(epoch) + VALIDATOR_PUBKEY + 0
    );

    [
        AssertCoinAnnouncement(
            sha256(CHECKPOINT_SINGLETON_ID + expected_announcement)
        ),
        CreateCoin(withdraw_delay_puzzle_hash, collateral_amount),
    ]
}
```

### Impact on Registration Coin Puzzle Hash

The registration coin now has **4 curried parameters** (up from 2):

```
registration_coin_puzzle_hash = curry_hash(
    REGISTRATION_COIN_MOD_HASH,
    validator_pubkey,
    checkpoint_singleton_id,
    WITHDRAW_DELAY_MOD_HASH,       // NEW
    WITHDRAW_DELAY_BLOCKS,          // NEW
)
```

This changes:
- `REGISTRATION_COIN_MOD_HASH` (the uncurried puzzle hash changes)
- All curry_hash computations that derive registration coin puzzle hashes
- The indexer's lineage verification
- The network coin's on-chain curry_hash

### Impact on Network Coin

The network coin puzzle source does NOT change, but the computed puzzle hash
for registration coins changes because the registration coin puzzle is
different. The network coin still uses:

```
CreateCoin(curry_hash(REGISTRATION_COIN_MOD_HASH, pubkey, ckpt_id), COLLATERAL_AMOUNT)
```

But `REGISTRATION_COIN_MOD_HASH` is now the tree hash of the updated
registration coin puzzle, and the network coin's on-chain curry must include
the two new parameters in its hash computation.

**Important:** The network coin's curry_hash call needs to include all 4
registration coin curried parameters. This means the network coin puzzle
itself needs updating to pass the two new parameters when computing the
registration coin puzzle hash.

### On-Chain curry_hash in Network Coin (Updated)

```rue
// In network_coin_inner.rue:
let registration_coin_puzzle_hash = curry_hash(
    REGISTRATION_COIN_MOD_HASH,
    new_validator_pubkey,
    CHECKPOINT_SINGLETON_ID,
    WITHDRAW_DELAY_MOD_HASH,       // NEW — must be curried into network coin too
    WITHDRAW_DELAY_BLOCKS,          // NEW — must be curried into network coin too
);
```

This means the network coin also gains two new curried parameters:
- `WITHDRAW_DELAY_MOD_HASH`
- `WITHDRAW_DELAY_BLOCKS`

### Cascade of Changes

| Component | Change |
|-----------|--------|
| `registration_coin.rue` | +2 curried params, CREATE_COIN target is delay coin hash |
| `network_coin_inner.rue` | +2 curried params for curry_hash computation |
| `REGISTRATION_COIN_MOD_HASH` | New value (puzzle changed) |
| `NETWORK_COIN_MOD_HASH` | New value (puzzle changed) |
| `NetworkConfig` | +2 fields (withdraw_delay_blocks, withdraw_delay_mod_hash) |
| Indexer lineage check | Updated puzzle hash computation |
| All tests computing puzzle hashes | Updated for new curry structure |

## Acceptance Criteria

- [ ] Registration coin has WITHDRAW_DELAY_MOD_HASH curried
- [ ] Registration coin has WITHDRAW_DELAY_BLOCKS curried
- [ ] Registration coin CREATE_COIN target is delay coin puzzle hash
- [ ] On-chain curry_hash matches Rust-computed delay coin puzzle hash
- [ ] Network coin updated with 2 additional curried params
- [ ] Registration coin no longer creates direct destination coin
