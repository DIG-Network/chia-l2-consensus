# WDC-010 — Destination Hint Memo

> **Authoritative requirement:** [WDC-010](../NORMATIVE.md#WDC-010)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [spec-withdraw-delay-coin.md](../../../../resources/spec-withdraw-delay-coin.md)

## Summary

The registration coin includes `collateral_destination` as a memo (hint) on
the CreateCoin that creates the withdraw delay coin. This enables
`get_coin_records_by_hint()` via `chia-query` for fast lookup of delay coins
by destination address. The withdraw delay coin includes a human-readable
memo `"DIG Network Collateral Release"` on its CreateCoin output.

## Specification

### Registration Coin Memo (Conflict-Resistant Hint)

When the registration coin creates the withdraw delay coin:

```rue
let hint: Bytes32 = sha256(CHECKPOINT_SINGLETON_ID + collateral_destination);
CreateCoin {
    puzzle_hash: withdraw_delay_puzzle_hash,
    amount: collateral_amount,
    memos: Memos { value: hint },  // WDC-010
}
```

The hint is `sha256(CHECKPOINT_SINGLETON_ID + collateral_destination)` (32 bytes).
Using the checkpoint singleton ID as a network prefix prevents cross-network
hint collisions. The indexer computes the same hash to query:
- `chia-query::get_coin_records_by_hint(hint_hex)` finds the delay coin
- The indexer can discover delay coins without scanning all coins by puzzle hash
- The L2 can quickly check if a validator has a pending withdrawal

### Withdraw Delay Coin Memo

When the delay coin releases funds:

```rue
CreateCoin {
    puzzle_hash: DESTINATION,
    amount: AMOUNT,
    memos: Memos { value: "DIG Network Collateral Release" },
}
```

This memo:
- Identifies the transaction in wallet UIs and block explorers
- Makes collateral releases searchable by memo text
- Has no security impact (informational only)

### Why Hint-Based Lookup

Without the hint:
1. To find a validator's delay coin, scan ALL coins with the delay coin
   puzzle hash prefix (expensive)
2. Decurry each one to check if its DESTINATION matches (very expensive)
3. No way to query by destination directly

With the hint:
1. Query `get_coin_records_by_hint(destination)` — returns only coins
   hinted with that destination
2. O(1) lookup, no decurrying needed
3. Same pattern as NET-005 (pubkey memo for registration coin detection)

## Acceptance Criteria

- [ ] Registration coin CreateCoin has collateral_destination as memo
- [ ] Memo is puzzle-enforced (in Rue source, not just driver convention)
- [ ] CLVM output includes the memo as 3rd element of CreateCoin condition
- [ ] Withdraw delay coin CreateCoin has "DIG Network Collateral Release" memo
- [ ] Both puzzles compile with memos
