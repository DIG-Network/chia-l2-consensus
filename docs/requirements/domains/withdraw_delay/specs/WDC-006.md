## Summary

`NetworkConfig` gains two fields for the withdraw delay mechanism. The delay
defaults to 24,000 blocks (~5 days at Chia's ~18-second block time) and is
fixed at deployment time.

## Specification

### New NetworkConfig Fields

```rust
pub struct NetworkConfig {
    // ... existing fields ...

    /// Number of L1 blocks a validator must wait after collateral recovery
    /// before funds are released. Default: 24,000 (~5 days).
    pub withdraw_delay_blocks: u64,

    /// Tree hash of the uncurried withdraw delay coin puzzle.
    pub withdraw_delay_mod_hash: Bytes32,
}
```

### Default Value

```rust
pub const DEFAULT_WITHDRAW_DELAY_BLOCKS: u64 = 24_000; // ~5 days at 18s/block
```

Calculation: `5 × 24 × 60 × 60 / 18 = 24,000 blocks`

### Immutability

Both fields are set at deployment and cannot change without redeploying the
entire network (network coin + checkpoint singleton + registration coin
puzzles all reference these values through currying).

### Serde

Both fields MUST be included in JSON serialization/deserialization of
`NetworkConfig`. The `withdraw_delay_mod_hash` is hex-encoded with `0x` prefix
per the existing convention.

## Acceptance Criteria

- [ ] `NetworkConfig` has `withdraw_delay_blocks: u64` field
- [ ] `NetworkConfig` has `withdraw_delay_mod_hash: Bytes32` field
- [ ] Default constant is 24,000 blocks
- [ ] JSON round-trip includes both fields
- [ ] Fields immutable after deployment
