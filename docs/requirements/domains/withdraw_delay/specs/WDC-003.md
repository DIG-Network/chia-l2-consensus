# WDC-003 — Fund Release

> **Authoritative requirement:** [WDC-003](../NORMATIVE.md#WDC-003)

> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

After the delay period, the withdraw delay coin creates a single coin at the
curried `DESTINATION` with the curried `AMOUNT`. This is the final step that
actually transfers collateral to the exiting validator's wallet.

## Specification

### CREATE_COIN Condition

```clvm
(CREATE_COIN DESTINATION AMOUNT)
```

Where:
- `DESTINATION`: 32-byte puzzle hash, curried at coin creation time
- `AMOUNT`: Integer in mojos, curried at coin creation time

### Amount Handling

The `AMOUNT` curried into the puzzle MUST equal the coin's actual amount.
This is guaranteed by the registration coin, which creates the withdraw delay
coin with `collateral_amount` as both the coin amount and the curried
`AMOUNT`.

If `AMOUNT > coin.amount`: spend fails (insufficient funds, Chia consensus
rejects).

If `AMOUNT < coin.amount`: the difference is burned as fees. The driver MUST
NOT set `AMOUNT < coin.amount`.

### Destination Flexibility

The `DESTINATION` can be any valid puzzle hash:
- Standard wallet (p2_delegated_puzzle_or_hidden_puzzle)
- Multi-sig wallet
- Another smart contract
- Governance/treasury address (for slashing)

The destination is set by the exiting validator when they spend the
registration coin (in the `collateral_destination` solution parameter). It
is then curried into the withdraw delay coin and cannot be changed.

### Puzzle Hash as Commitment

The withdraw delay coin's puzzle hash is a cryptographic commitment to
(destination, amount, delay):

```
puzzle_hash = curry_hash(
    WITHDRAW_DELAY_MOD_HASH,
    destination,
    amount,
    delay_blocks,
)
```

This means:
- The registration coin can verify the withdraw delay coin's behavior by
  computing its puzzle hash
- An observer can verify what a withdraw delay coin will do by decurrying it
- No party can alter the destination or amount after creation

## Acceptance Criteria

- [ ] CLVM output contains CREATE_COIN with correct DESTINATION and AMOUNT
- [ ] DESTINATION matches curried value exactly
- [ ] AMOUNT matches curried value exactly
- [ ] Different destinations produce different puzzle hashes
- [ ] Different amounts produce different puzzle hashes
- [ ] Rust-computed puzzle hash matches CLVM curry_hash
