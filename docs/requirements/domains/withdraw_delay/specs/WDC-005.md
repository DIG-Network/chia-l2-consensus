## Summary

The crate provides `release_collateral()` on `ConsensusClient` to build a
SpendBundle that spends a withdraw delay coin after the delay period. Per
API-008, the method returns the bundle without broadcasting it.

## Specification

### New Public Method

```rust
impl ConsensusClient {
    /// Build a spend bundle to release collateral from a withdraw delay coin.
    /// Returns SpendBundle — the caller broadcasts it.
    ///
    /// Two-phase collateral recovery:
    ///   1. recover_collateral() — creates the withdraw delay coin
    ///   2. [wait withdraw_delay_blocks L1 blocks]
    ///   3. release_collateral() — releases funds to destination
    ///
    /// If the delay has not elapsed, the Chia node will reject the bundle.
    /// CLVM cost: ~2.4M units
    pub async fn release_collateral(
        &self,
        withdraw_delay_coin: Coin,
        destination: Bytes32,
        amount: u64,
    ) -> ConsensusResult<SpendBundle>;
}
```

### Updated recover_collateral()

The existing method signature is unchanged but the output changes: the
registration coin spend now creates a withdraw delay coin instead of a
direct destination coin.

The method's doc comment should be updated to explain the two-phase flow:

```rust
/// Build a spend bundle to initiate collateral recovery.
/// Returns SpendBundle — the caller broadcasts it.
///
/// This creates a WITHDRAW DELAY COIN that holds the collateral
/// for withdraw_delay_blocks L1 blocks. After the delay, call
/// release_collateral() to send the funds to the destination.
pub async fn recover_collateral(
    &self,
    validator_pubkey: PublicKey,
    collateral_destination: Bytes32,
) -> ConsensusResult<SpendBundle>;
```

### Return Value (API-008 Compliance)

Both `recover_collateral()` and `release_collateral()` return `SpendBundle`.
Neither method broadcasts. The caller is responsible for:
1. Broadcasting the recovery bundle
2. Waiting for confirmation + delay period
3. Broadcasting the release bundle
