# WDC-008 — CLVM Execution Tests

> **Authoritative requirement:** [WDC-008](../NORMATIVE.md#WDC-008)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

CLVM execution tests deserialize the compiled `.hex` artifact, curry with
test parameters, run via `run_program()`, and assert exact output conditions.
This follows SCHEMA.md Hard Testing Requirements.

## Required Tests

### WDC-001 (Puzzle Structure)

| Test | What it verifies |
|------|-----------------|
| `vv_req_wdc_008_clvm_loads_and_executes` | .hex deserializes, curried puzzle runs |
| `vv_req_wdc_008_clvm_exactly_two_conditions` | Output has 2 conditions |
| `vv_req_wdc_008_clvm_no_passthrough` | Empty solution, no extra conditions |
| `vv_req_wdc_008_clvm_different_params_different_hash` | Varying destination/amount/delay |

### WDC-002 (Time Lock)

| Test | What it verifies |
|------|-----------------|
| `vv_req_wdc_008_clvm_assert_height_relative_present` | Opcode 82 in output |
| `vv_req_wdc_008_clvm_delay_value_matches_curried` | Height relative value == WITHDRAW_DELAY_BLOCKS |
| `vv_req_wdc_008_clvm_delay_zero_valid` | Delay 0 produces valid output |

### WDC-003 (Fund Release)

| Test | What it verifies |
|------|-----------------|
| `vv_req_wdc_008_clvm_create_coin_destination` | CREATE_COIN puzzle hash == curried DESTINATION |
| `vv_req_wdc_008_clvm_create_coin_amount` | CREATE_COIN amount == curried AMOUNT |
| `vv_req_wdc_008_clvm_cross_impl_puzzle_hash` | Rust curry_hash == CLVM curry_hash for same inputs |

## Acceptance Criteria

- [ ] At least 10 CLVM execution tests
- [ ] All tests use compiled .hex artifact (not source inspection)
- [ ] Tests cover all 3 curried parameter variations
- [ ] Cross-implementation hash verification
- [ ] Tests verify exact condition opcodes and values
