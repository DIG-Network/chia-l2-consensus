## Summary

The system assumes a majority of validators (>50%) are honest. If this assumption is violated, fraudulent checkpoints can be submitted and the system provides no safety guarantees. This is the fundamental assumption of any BFT consensus system.

## Specification

### The Assumption

The system is secure as long as fewer than half of the registered validators collude to produce fraudulent checkpoints. A coalition of exactly half or more validators can submit any state root they want.

### What Majority Enables

With honest majority:
- The correct `validator_merkle_root` is always signed
- Fraudulent state roots cannot gather sufficient signatures
- Collateral provides economic security against attacks

### What Happens If Violated

If majority is compromised:
- Any arbitrary state can be checkpointed
- Collateral recovery for honest validators may be blocked
- L2 state integrity is lost

### Mitigation Strategies

| Strategy | Effect |
|----------|--------|
| Collateral requirements | Makes attacking expensive |
| Economic independence | Reduces collusion risk |
| Geographic distribution | Reduces coordination risk |
| Diverse validator set | Increases attack complexity |

## Acceptance Criteria

- [ ] Documentation clearly states majority assumption
- [ ] Validator onboarding explains the trust model
- [ ] Monitoring alerts on validator set changes
- [ ] Governance can respond to majority attacks

## Implementation Notes

This is a design assumption, not a code implementation. The assumption is enforced through:
- Circuit constraint requiring 2k > validator_count
- Economic incentives via collateral
- Social coordination of validator selection

### Circuit Enforcement

```rust
// Constraint 3: Majority threshold
// Proves: 2k > validator_count
enforce!(2 * signer_count > validator_count);
```

## Verification

1. Verify circuit enforces strict majority (>50%, not >=50%)
2. Verify checkpoint requires majority signature
3. Document attack scenarios under majority violation
4. Establish governance response procedures
