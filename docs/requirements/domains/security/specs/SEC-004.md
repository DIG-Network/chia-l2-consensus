# SEC-004 — Trusted Setup

> **Authoritative requirement:** [SEC-004](../NORMATIVE.md#SEC-004)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **CHIP reference:** [chip-groth16-l2-consensus.md](../../../../resources/chip-groth16-l2-consensus.md) — Security

## Summary

The Groth16 trusted setup must be performed as a multi-party computation ceremony. At least one honest participant is required for soundness. Single-party setups MUST NOT be used in production.

## Specification

### The Assumption

If any participant in the MPC trusted setup ceremony honestly destroys their contribution (the "toxic waste"), the setup is secure. If all participants collude, they can generate fake proofs.

### Single-Party Setup

- Acceptable only for development and testing
- Never use in production
- One person controls all toxic waste
- Can generate arbitrary fake proofs

### Multi-Party Ceremony

| Requirement | Purpose |
|-------------|---------|
| Multiple participants | Distributes trust |
| Diverse backgrounds | Reduces coordination risk |
| Geographic distribution | Prevents physical coercion |
| Public attestations | Accountability |
| Published transcript | Verifiability |

### Compromise Recovery

If trusted setup is compromised:
1. Detect via fraudulent checkpoints
2. Rerun MPC ceremony with new participants
3. Redeploy checkpoint singleton with new VK
4. Coordinate L2 migration

## Implementation Notes

- **Setup tool:** `l2-consensus-setup`
- **Phase 1:** Powers of Tau (can reuse Hermez)
- **Phase 2:** Circuit-specific MPC

### Ceremony Options

**Using snarkjs:**
```bash
snarkjs groth16 setup circuit.r1cs powersOfTau28_hez_final.ptau ceremony_0000.zkey
snarkjs zkey contribute ceremony_0000.zkey ceremony_0001.zkey --name "Participant 1"
# ... more participants
snarkjs zkey beacon ceremony_final.zkey final.zkey <BEACON> "Final" 10
```

**Using Arkworks:**
```bash
ark-mpc-cli phase2 init --circuit circuit.r1cs --ptau tau.ptau --output init.params
ark-mpc-cli phase2 contribute --input init.params --output contrib.params
ark-mpc-cli phase2 finalize --input final.params --pk-output pk.bin --vk-output vk.bin
```

## Verification

1. Verify ceremony transcript is complete
2. Verify each contribution is valid
3. Verify participant attestations
4. Verify VK matches published hash
5. Verify deployed singleton uses correct VK
