# SEC-003 — Collateral Security

> **Authoritative requirement:** [SEC-003](../NORMATIVE.md#SEC-003)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **CHIP reference:** [chip-groth16-l2-consensus.md](../../../../resources/chip-groth16-l2-consensus.md) — Security

## Summary

A validator cannot recover their collateral without a checkpoint that excludes them from the active set, as verified by a non-membership proof. This prevents validators from abandoning the network unilaterally.

## Specification

### Collateral Lock Mechanism

1. Collateral is locked in the registration coin at registration
2. Registration coin can only be spent when checkpoint singleton emits non-membership announcement
3. Non-membership requires Merkle proof against current `validator_merkle_root`
4. Merkle root only changes via checkpoint with majority signature

### Why This Design

| Goal | How Achieved |
|------|--------------|
| Prevent premature exit | Collateral locked until exclusion confirmed |
| Enable voluntary exit | Signal at L2, wait for exclusion checkpoint |
| Enable forced exit | Majority can exclude any validator |
| Enable slashing | Collateral recovery can be redirected |

### Recovery Process

1. Validator signals exit intent at L2 level
2. Next checkpoint excludes validator from Merkle root
3. Validator generates non-membership proof
4. Membership query spend emits non-membership announcement
5. Registration coin spend asserts announcement and returns collateral

### What Collateral Cannot Do

- Cannot be recovered without majority consensus on exclusion
- Cannot be recovered if validator loses wallet access
- Cannot bypass the checkpoint requirement

## Acceptance Criteria

- [ ] Registration coin requires non-membership announcement to spend
- [ ] Non-membership requires valid Merkle proof
- [ ] Merkle root only updates via valid checkpoint
- [ ] Collateral amount matches network configuration
- [ ] Documentation explains recovery process and timing

## Implementation Notes

- **Lock:** `puzzles/registration_coin.clsp`
- **Unlock:** Membership query spend + registration coin spend
- **Proof:** Non-membership Merkle proof

### Registration Coin Spend Conditions

```clojure
; Must assert non-membership announcement from checkpoint singleton
(AssertCoinAnnouncement
    (sha256
        checkpoint_singleton_coin_id
        (membership_announcement epoch pubkey 0x00)  ; is_member = false
    )
)
```

## Verification

1. Attempt to spend registration coin without announcement -> fails
2. Attempt with membership announcement (is_member=true) -> fails
3. Attempt with non-membership from wrong epoch -> fails
4. Proper non-membership announcement -> succeeds
5. Verify collateral returns to specified destination
