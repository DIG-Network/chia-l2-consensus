# SEC-002 — Two-Check Completeness

> **Authoritative requirement:** [SEC-002](../NORMATIVE.md#SEC-002)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **CHIP reference:** [chip-groth16-l2-consensus.md](../../../../resources/chip-groth16-l2-consensus.md) — Security

## Summary

The Groth16 proof verification and BLS signature verification together provide complete security. Neither check alone is sufficient to prevent fraudulent checkpoints.

## Specification

### The Two Checks

| Check | What It Proves | Not Sufficient Alone Because |
|-------|----------------|------------------------------|
| Groth16 proof | k pubkeys are members, sum to agg_signers, k is majority | Does not prove those pubkeys actually signed |
| BLS signature | agg_signers signed the checkpoint message | Does not prove agg_signers is a legitimate majority |

### Why Both Are Required

**ZK proof alone:**
- Proves the aggregate pubkey represents a majority
- Does NOT prove those validators actually signed
- Attacker could use valid pubkeys without signatures

**BLS signature alone:**
- Proves the aggregate pubkey signed the message
- Does NOT prove the aggregate represents a majority
- Single validator could claim to be majority

### Combined Security

Together they prove:
1. The signing pubkeys are all registered validators (ZK: Merkle membership)
2. The signing pubkeys form a majority (ZK: 2k > count)
3. Those exact pubkeys produced the aggregate (ZK: sum = agg_signers)
4. That aggregate actually signed the checkpoint (BLS: signature valid)

## Acceptance Criteria

- [ ] Checkpoint singleton verifies Groth16 proof
- [ ] Checkpoint singleton verifies BLS signature
- [ ] Both checks use the same agg_signers value
- [ ] Either check failing rejects the checkpoint
- [ ] Documentation explains the completeness argument

## Implementation Notes

- **On-chain:** `puzzles/checkpoint_inner.clsp`
- **ZK verification:** `bls_pairing_identity`
- **BLS verification:** `bls_verify`

### Checkpoint Singleton Logic

```clojure
; Both checks must pass
(let
    (zk_valid (verify_groth16_proof proof vk_input agg_signers ...))
    (sig_valid (bls_verify agg_signers agg_sig checkpoint_message))
)
(assert (and zk_valid sig_valid))
```

## Verification

1. Submit checkpoint with valid proof, invalid signature -> rejected
2. Submit checkpoint with invalid proof, valid signature -> rejected
3. Submit checkpoint with mismatched agg_signers -> rejected
4. Submit checkpoint with both valid -> accepted
5. Formal analysis of completeness
