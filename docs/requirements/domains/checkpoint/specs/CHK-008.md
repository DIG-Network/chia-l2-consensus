## Summary

A full end-to-end integration test exercises every component of the system in a single lifecycle. This is the definitive proof that all puzzles, provers, and drivers work together correctly. It uses the chia-wallet-sdk simulator for full consensus validation including CLVM execution, signature aggregation, and announcement matching.

## Specification

### Test Lifecycle

```
1. DEPLOY
   - Run trusted setup → proving key + verification key
   - Deploy network coin + checkpoint singleton in one bundle via simulator
   - Verify both singletons exist on-chain

2. REGISTER (repeat for k validators)
   - Generate BLS keypair
   - Spend network coin to register → registration coin created
   - Verify indexer detects registration coin with valid lineage

3. CHECKPOINT
   - Build sparse Merkle tree from registered validators
   - Compute checkpoint message (new state root, new merkle root, new count, new epoch)
   - Collect k validator signatures over checkpoint message
   - Run Groth16 prover with real circuit → 192-byte proof
   - Submit checkpoint spend to simulator:
     * bls_pairing_identity verifies Groth16 proof
     * bls_verify verifies aggregate signature
     * Epoch increments, state updates
   - Verify simulator accepted the spend
   - Verify new checkpoint singleton has updated state

4. EXIT + COLLATERAL RECOVERY
   - Next checkpoint excludes one validator
   - Validator queries membership (non-membership proof)
   - Checkpoint singleton emits non-membership announcement
   - Registration coin spend asserts that announcement
   - Same spend bundle: membership query + registration coin spend
   - Verify collateral returned to validator

5. FAILURE CASES
   - Submit checkpoint with invalid proof → rejected
   - Submit checkpoint with wrong signature → rejected
   - Submit checkpoint with minority signers → proof generation fails
   - Attempt collateral recovery without membership query → rejected
   - Attempt collateral recovery with stale epoch → rejected
```

### Required Infrastructure

| Component | Crate | Usage |
|-----------|-------|-------|
| Simulator | `chia-sdk-test` | Full consensus validation |
| Spend building | `chia-sdk-driver` | SpendContext, StandardLayer, Singleton |
| CLVM runtime | `clvmr` | Puzzle execution |
| BLS signing | `blst` | Key generation, signing |
| Groth16 prover | `ark-groth16` | Proof generation |
| Sparse Merkle Tree | `src/merkle/` | Validator set |
| Wire format | `src/prover/serialize.rs` | Message hashing |

### Test Duration

Expected: 5-15 minutes per run (dominated by Groth16 proof generation).
For CI: use a small circuit (MAX_SIGNERS=4, TREE_DEPTH=4) to reduce proof time.

## Acceptance Criteria

- [ ] Trusted setup produces valid proving key + verification key
- [ ] Network coin + checkpoint singleton deployed via simulator
- [ ] At least 3 validators registered successfully
- [ ] Groth16 proof generated with real circuit and real witnesses
- [ ] Checkpoint spend accepted by simulator with valid proof + signature
- [ ] Epoch increments and state updates verified
- [ ] Collateral recovery succeeds after validator exit
- [ ] Invalid proof rejected by simulator
- [ ] Invalid signature rejected by simulator
- [ ] Cross-coin announcement matching works (checkpoint → registration)

## Implementation Notes

- **Test file:** `tests/integration_e2e.rs`
- **Dependencies:** All crates in dev-dependencies
- **CI configuration:** Use `#[ignore]` for slow tests, run in nightly CI

### Simulator Usage Pattern

```rust
use chia_sdk_test::Simulator;
use chia_sdk_driver::SpendContext;

let mut sim = Simulator::new();

// Deploy singletons
let (network_coin, checkpoint) = deploy(&mut sim, &config);

// Register validators
for sk in &validator_keys {
    register_validator(&mut sim, &network_coin, sk);
}

// Submit checkpoint
let proof = generate_groth16_proof(&circuit, &witnesses);
let bundle = build_checkpoint_bundle(&checkpoint, &proof, &agg_sig);
sim.new_transaction(bundle, &TESTNET11_CONSTANTS)?;

// Verify state
let new_state = sim.coin_state(new_checkpoint_id);
assert_eq!(new_state.epoch, 1);
```
