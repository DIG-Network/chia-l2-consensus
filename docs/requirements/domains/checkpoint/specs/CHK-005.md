# CHK-005 — Membership Query

## Summary

The membership query spend path allows anyone to query whether a validator is currently in the active set. It verifies a Merkle proof, emits an announcement with the result, and recreates the singleton unchanged. This enables registration coins to verify non-membership for collateral recovery.

## Specification

### Query Types

| Query | Proof Type | Result |
|-------|-----------|--------|
| Is member? | Membership proof | Leaf = sha256(pubkey) |
| Is not member? | Non-membership proof | Leaf = EMPTY_LEAF_HASH |

### Merkle Proof Verification

The puzzle verifies the Merkle path from leaf to root:

```clojure
(defun verify_merkle_path (leaf siblings index depth)
    (if (= depth 0)
        leaf  ; Return computed root
        (let ((sibling (first siblings))
              (is_left (= (logand index 1) 0)))
            (verify_merkle_path
                (if is_left
                    (sha256 leaf sibling)    ; leaf on left
                    (sha256 sibling leaf))   ; leaf on right
                (rest siblings)
                (lsh index -1)
                (- depth 1)))))

; Verify computed root equals state root
(= (verify_merkle_path leaf siblings index TREE_DEPTH)
   VALIDATOR_MERKLE_ROOT)
```

### Membership Query

To prove pubkey IS a member:
```
leaf = sha256(pubkey)        ; Active leaf value
siblings = [32 siblings]      ; Merkle path
index = slot(pubkey)          ; Deterministic from pubkey

verify_merkle_path(leaf, siblings, index, TREE_DEPTH) == VALIDATOR_MERKLE_ROOT
```

### Non-Membership Query

To prove pubkey is NOT a member:
```
leaf = EMPTY_LEAF_HASH        ; Empty slot marker
siblings = [32 siblings]      ; Merkle path
index = slot(pubkey)          ; Where pubkey WOULD be

verify_merkle_path(leaf, siblings, index, TREE_DEPTH) == VALIDATOR_MERKLE_ROOT
```

### Announcement Format

After verification, emit announcement:

```clojure
(CREATE_COIN_ANNOUNCEMENT
    (sha256
        "membership"          ; 10 bytes
        epoch_be8             ; 8 bytes, current epoch
        query_pubkey          ; 48 bytes
        is_member))           ; 1 byte: 0x01 or 0x00
```

### Singleton Recreation (Unchanged)

The membership query recreates the singleton with identical state:

```clojure
(CREATE_COIN
    MY_PUZZLE_HASH            ; Same puzzle hash (same state)
    MY_AMOUNT)                ; 1 mojo
```

No state changes occur — epoch, merkle root, count all preserved.

### Solution Parameters

```clojure
; Membership query spend path solution
spend_path = 1                ; Indicates membership query
query_pubkey                  ; 48 bytes
is_member                     ; boolean
merkle_siblings               ; List of 32-byte hashes, length = TREE_DEPTH
```

### Validation Logic

```clojure
(if (= is_member 1)
    ; Membership query - leaf is hash of pubkey
    (verify_merkle_path (sha256 query_pubkey) ...)
    ; Non-membership query - leaf is empty
    (verify_merkle_path EMPTY_LEAF_HASH ...))
```

### Use Case: Collateral Recovery

```
1. Validator is excluded from checkpoint (no longer in Merkle tree)
2. Validator builds non-membership proof:
   - Find their slot in the tree
   - Collect sibling hashes from empty slot to root
3. Spend checkpoint singleton (membership query):
   - Solution: (1, pubkey, false, [siblings])
   - Verification: empty leaf at slot produces current root
   - Announcement: "membership" + epoch + pubkey + 0x00
4. Same bundle: spend registration coin
   - Asserts the non-membership announcement
   - Recovers collateral
```

## Acceptance Criteria

- [ ] Membership proof verified: sha256(pubkey) at slot → root
- [ ] Non-membership proof verified: EMPTY_LEAF at slot → root
- [ ] Announcement emitted with correct format
- [ ] Announcement includes current epoch
- [ ] Announcement includes is_member byte
- [ ] Singleton recreated unchanged (same puzzle hash)
- [ ] Invalid Merkle proof → spend fails
- [ ] Wrong is_member value → spend fails

## Verification

1. Register validator; verify in Merkle tree
2. Build membership proof for validator
3. Submit membership query (is_member=true)
4. Verify announcement: pubkey + 0x01
5. Exit validator via checkpoint
6. Build non-membership proof
7. Submit membership query (is_member=false)
8. Verify announcement: pubkey + 0x00
9. Verify singleton state unchanged both times
10. Submit with invalid proof → verify failure
