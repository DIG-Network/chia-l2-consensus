# DEP-003 — Initial State

> **Authoritative requirement:** [DEP-003](../NORMATIVE.md#DEP-003)

> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

The checkpoint singleton must be deployed with specific initial state: epoch=0, validator_count=0, validator_merkle_root=EMPTY_TREE_ROOT, and state_root as application-defined genesis.

## Specification

### Initial State Values

| Field | Initial Value | Rationale |
|-------|---------------|-----------|
| `epoch` | 0 | Epochs start from zero |
| `validator_count` | 0 | No validators registered yet |
| `validator_merkle_root` | EMPTY_TREE_ROOT | Empty sparse Merkle tree root |
| `state_root` | Application-defined | L2 genesis state |

### EMPTY_TREE_ROOT

The empty tree root is computed by recursively hashing empty leaves:

```rust
const EMPTY_LEAF_HASH: [u8; 32] = sha256(b"EMPTY_LEAF");

fn empty_tree_root(depth: u32) -> [u8; 32] {
    if depth == 0 {
        return EMPTY_LEAF_HASH;
    }
    let child = empty_tree_root(depth - 1);
    sha256(&[child, child].concat())
}
```

### State Root

The `state_root` is application-specific and defined by the L2 system:
- Could be hash of genesis block
- Could be predefined constant
- Must be documented for the specific L2

## Acceptance Criteria

- [ ] Epoch is initialized to 0
- [ ] Validator count is initialized to 0
- [ ] Merkle root matches EMPTY_TREE_ROOT for configured depth
- [ ] State root is documented application genesis
- [ ] All values are correctly curried into singleton

## Implementation Notes

- **Computation:** `src/deploy/initial_state.rs`
- **Currying:** `src/checkpoint/deploy.rs`

### Computing Initial State

```rust
pub fn initial_checkpoint_state(
    tree_depth: u32,
    genesis_state_root: [u8; 32],
) -> CheckpointState {
    CheckpointState {
        epoch: 0,
        validator_count: 0,
        validator_merkle_root: empty_tree_root(tree_depth),
        state_root: genesis_state_root,
    }
}
```

### Deployment Configuration

```json
{
  "tree_depth": 32,
  "genesis_state_root": "0x...",  // Application-defined
  "empty_leaf_hash": "0x...",      // Precomputed
  "empty_tree_root": "0x..."       // For tree_depth
}
```

## Verification

1. Compute EMPTY_TREE_ROOT for configured depth
2. Deploy checkpoint singleton
3. Decurry and verify epoch = 0
4. Verify validator_count = 0
5. Verify merkle_root = EMPTY_TREE_ROOT
6. Verify state_root matches genesis config
