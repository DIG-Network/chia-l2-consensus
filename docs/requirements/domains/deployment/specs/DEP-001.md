# DEP-001 — Trusted Setup

> **Authoritative requirement:** [DEP-001](../NORMATIVE.md#DEP-001)

> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

Before deployment, a Groth16 trusted setup ceremony must be performed to generate the proving key and verification key bound to the circuit parameters (MAX_SIGNERS, TREE_DEPTH).

## Specification

### What Must Be Generated

| Artifact | Size | Purpose |
|----------|------|---------|
| Proving key (pk) | 100-500 MB | Off-chain proof generation |
| Verification key (vk) | 672 bytes | On-chain proof verification, curried into singleton |

### Circuit Parameters

The setup is bound to specific circuit parameters that cannot change after deployment:
- `MAX_SIGNERS`: Maximum number of validators that can sign
- `TREE_DEPTH`: Depth of the sparse Merkle tree

### Ceremony Requirements

| Requirement | Reason |
|-------------|--------|
| Multi-party computation | Single-party is insecure |
| At least 5 participants | Distributes trust |
| Geographic diversity | Prevents coordination |
| Published transcript | Enables verification |
| Participant attestations | Accountability |

### Pre-Deployment Checklist

1. [ ] Ceremony completed with MPC
2. [ ] Transcript published
3. [ ] VK hash computed and published
4. [ ] Test proof generated and verified
5. [ ] Cross-verification Rust/CLVM passed

## Acceptance Criteria

- [ ] Trusted setup completed before deployment
- [ ] Ceremony uses MPC (not single-party)
- [ ] Circuit parameters match deployment config
- [ ] VK has correct structure (7 IC points)
- [ ] Test proof verifies in both Rust and CLVM

## Implementation Notes

- **Tool:** `l2-consensus-setup`
- **Phase 1:** Powers of Tau (can reuse Hermez)
- **Phase 2:** Circuit-specific setup

### Generating Keys

```bash
cargo build --release --bin l2-consensus-setup

./target/release/l2-consensus-setup \
  --max-signers 64 \
  --tree-depth 32 \
  --pk-path proving_key.bin \
  --vk-path verification_key.bin
```

### Verifying Output

```rust
let vk = load_verification_key("verification_key.bin")?;
assert_eq!(vk.gamma_abc_g1.len(), 7, "Must have 7 IC points");

let test_proof = generate_test_proof(&pk, MAX_SIGNERS, TREE_DEPTH)?;
assert!(Groth16::<Bls12_381>::verify(&vk, &inputs, &proof)?);
```

## Verification

1. Verify ceremony transcript is valid
2. Verify VK has correct structure
3. Generate and verify test proof
4. Cross-verify in CLVM simulator
5. Compute and record VK hash
