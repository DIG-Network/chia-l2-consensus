# DEP-005 — Artifact Publication

> **Authoritative requirement:** [DEP-005](../NORMATIVE.md#DEP-005)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **CHIP reference:** [chip-groth16-l2-consensus.md](../../../../resources/chip-groth16-l2-consensus.md) — Deployment

## Summary

Deployment artifacts must be published: network_config.json, verification key (hex), VK hash, ceremony transcript, and circuit source code for independent verification.

## Specification

### Required Artifacts

| Artifact | Format | Purpose |
|----------|--------|---------|
| `network_config.json` | JSON | Network parameters for validators |
| Verification key | Hex-encoded JSON | On-chain verification reference |
| VK hash | 32-byte hex | Quick VK verification |
| Ceremony transcript | `.zkey` or `.params` files | Ceremony verification |
| Circuit source code | Git commit | Independent reproduction |

### network_config.json

```json
{
  "network_coin_launcher_id": "0x...",
  "checkpoint_launcher_id": "0x...",
  "genesis_challenge": "0x...",
  "collateral_amount": 10000000000000,
  "tree_depth": 32,
  "empty_leaf_hash": "0x...",
  "registration_coin_mod_hash": "0x...",
  "vk_hash": "0x..."
}
```

### Verification Key Format

```json
{
  "alpha_g1": "0xa305...",
  "beta_g2": "0xa624...",
  "gamma_g2": "0xb123...",
  "delta_g2": "0xc567...",
  "ic": [
    "0xa111...",
    "0xa222...",
    "0xa333...",
    "0xa444...",
    "0xa555...",
    "0xa666...",
    "0xa777..."
  ]
}
```

### Publication Locations

- Official repository release
- IPFS with pinned CID
- Project documentation site
- At least two independent mirrors

## Acceptance Criteria

- [x] network_config.json published with all parameters (`vv_req_dep_005_all_fields_present`)
- [x] VK published in hex-encoded JSON format (`vv_req_dep_005_vk_json_structure`)
- [x] VK hash matches SHA-256 of VK content (`vv_req_dep_005_vk_hash_matches_vk_content`)
- [x] JSON round-trips correctly (`vv_req_dep_005_json_roundtrip`)
- [x] Hex fields use 0x prefix, Bytes32 fields are 66 chars (`vv_req_dep_005_hex_fields_have_prefix`, `vv_req_dep_005_bytes32_fields_correct_length`)
- [ ] Ceremony transcript published and downloadable (process — not code-testable)
- [ ] Circuit source at exact commit published (process — not code-testable)

## Implementation Notes

- **Generator:** `l2-consensus-deploy --publish-artifacts`
- **Output directory:** `./deployment-artifacts/`

### Artifact Generation

```bash
./target/release/l2-consensus-deploy \
  --genesis-coin <coin_id> \
  --vk-path verification_key.bin \
  --publish-artifacts ./deployment-artifacts/

ls ./deployment-artifacts/
# network_config.json
# verification_key.json
# vk_hash.txt
# circuit_source_commit.txt
```

### Verification Steps for Validators

1. Download network_config.json
2. Download verification_key.json
3. Compute sha256(verification_key.json)
4. Verify hash matches vk_hash in config
5. Verify deployed singleton VK matches

## Verification

1. Generate all deployment artifacts
2. Verify network_config.json has all required fields
3. Verify VK hash computation is correct
4. Verify artifacts are accessible from published locations
5. Verify a new validator can onboard using published artifacts
