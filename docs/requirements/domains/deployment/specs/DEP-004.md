# DEP-004 — Verification Key Verification

> **Authoritative requirement:** [DEP-004](../NORMATIVE.md#DEP-004)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **CHIP reference:** [chip-groth16-l2-consensus.md](../../../../resources/chip-groth16-l2-consensus.md) — Security

## Summary

After deployment, operators must verify the VK curried into the checkpoint singleton matches the expected VK from the trusted setup ceremony by decurrying and comparing.

## Specification

### Why Verification Matters

The VK is cryptographically bound to the circuit. A wrong VK means:
- Proofs for the wrong circuit would verify
- Security assumptions may not hold
- Trusted setup guarantees don't apply

### Verification Process

1. **Fetch deployed singleton puzzle**
2. **Decurry to extract VK**
3. **Compare against published VK hash**
4. **Verify VK structure**

### VK Hash

The VK hash is SHA-256 of the serialized verification key. This short value (32 bytes) uniquely identifies the VK and is published alongside the ceremony transcript.

### What to Verify

| Check | How |
|-------|-----|
| VK hash matches | SHA-256(serialized_vk) = published_hash |
| IC point count | vk.ic.len() == 7 |
| Point sizes | G1 = 48 bytes, G2 = 96 bytes |
| Alpha, beta, gamma, delta present | All required points exist |

## Acceptance Criteria

- [x] VK can be extracted from deployed singleton — `extract_vk_components_from_bytes()` (`vv_req_dep_004_components_from_bytes_match`)
- [x] Extracted VK hash matches published hash — `verify_vk_hash()` (`vv_req_dep_004_hash_matches`)
- [x] VK has 7 IC points — `validate_vk_bytes()` (`vv_req_dep_004_validate_bytes_succeeds`)
- [x] All point sizes are correct — 672-byte validation (`vv_req_dep_004_vk_byte_size_is_672`)
- [x] Verification functions documented and available (`vv_req_dep_004_full_verification_workflow`)

## Implementation Notes

- **Tool:** `l2-consensus-verify`
- **Input:** Checkpoint singleton launcher ID, published VK hash
- **Output:** PASS/FAIL with details

### Verification Tool

```bash
./target/release/l2-consensus-verify \
  --checkpoint-launcher-id <launcher_id> \
  --expected-vk-hash <published_hash> \
  --node-url https://your-node:8555
```

### Verification Code

```rust
pub fn verify_deployed_vk(
    singleton_puzzle: &Program,
    expected_vk_hash: [u8; 32],
) -> Result<bool, VerificationError> {
    // Decurry to extract VK
    let (_, curried_args) = singleton_puzzle.uncurry()?;
    let vk = extract_vk_from_args(&curried_args)?;

    // Verify structure
    assert_eq!(vk.ic.len(), 7, "VK must have 7 IC points");
    assert_eq!(vk.alpha_g1.len(), 48);
    assert_eq!(vk.beta_g2.len(), 96);

    // Compare hash
    let vk_hash = sha256(&serialize_vk(&vk));
    Ok(vk_hash == expected_vk_hash)
}
```
