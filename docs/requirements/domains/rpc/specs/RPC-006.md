# RPC-006 — Dependency Version Alignment

> **Authoritative requirement:** [RPC-006](../NORMATIVE.md#RPC-006)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

Adding `chia-query` and `dig-l1-wallet` requires aligning the Chia ecosystem
crate versions. Both upstream crates use newer versions of `chia-wallet-sdk`
and related crates than our project currently uses. Types from these crates
(e.g., `Coin`, `Bytes32`, `SpendBundle`) must be the same type across crate
boundaries to avoid compilation errors.

## Specification

### Breaking Changes to Handle

The version bump from `chia-wallet-sdk 0.18` → `0.30` will likely introduce
API changes. Common areas:

| Area | Likely Changes |
|------|---------------|
| `SpendContext` | Method signatures may differ |
| `Launcher` | Singleton creation API may change |
| `CurriedProgram` | Currying API updates |
| `Coin` type | Now from `chia` umbrella crate |
| `Bytes32` | Import path changes |
| `singleton_top_layer_v1_1` | May move or rename |

### Verification Strategy

```bash
# 1. Update Cargo.toml with new versions
# 2. Run cargo check to find all type mismatches
cargo check 2>&1 | grep "error"
# 3. Fix each mismatch (mostly import path changes)
# 4. Verify tests still compile
cargo check --tests
# 5. Run existing tests to verify no behavioral changes
cargo test
```

### Type Interoperability

After alignment, these types MUST be the same across all crates:

| Type | Used In |
|------|---------|
| `Coin` | chia-query returns, chia-wallet-sdk consumes, our puzzles use |
| `Bytes32` | Everywhere — puzzle hashes, coin IDs, roots |
| `SpendBundle` | chia-query push_tx, our driver output |
| `PublicKey` | BLS pubkeys across prover and puzzle drivers |
| `CoinSpend` | chia-query returns, our drivers produce |

If types differ across crate versions, the compiler will emit E0308 (type
mismatch) errors. Fix by ensuring all crates use the same version of the
underlying type crate.
