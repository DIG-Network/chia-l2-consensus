## Summary

The 5 `todo!()` stubs in `src/client.rs` are the public facade methods that
coordinate puzzle drivers, prover, and indexer. Each method follows the
pattern: ensure synced → validate preconditions → build SpendBundle → return.

## Stubs to Implement

| Method | What It Builds | Key Dependencies |
|--------|---------------|-----------------|
| `deploy()` | Network coin + checkpoint singleton launchers | SpendContext, genesis coin |
| `register_validator()` | Network coin spend → registration coin | Indexer state, coin selection (RPC-005) |
| `build_checkpoint()` | Checkpoint spend with Groth16 + BLS | Prover, indexer state, SpendContext |
| `get_checkpoint_state()` | *(no bundle — reads cached state)* | Indexer cache |
| `recover_collateral()` | Membership query + registration coin spend | Indexer state, Merkle proofs |
