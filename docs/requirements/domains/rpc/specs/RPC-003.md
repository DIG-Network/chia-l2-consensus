# RPC-003 — Indexer Sync Algorithm

> **Authoritative requirement:** [RPC-003](../NORMATIVE.md#RPC-003)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

The indexer sync algorithm uses `chia-query::ChiaQuery` to discover on-chain
state: network coin position, checkpoint singleton state, and all valid
registration coins. It verifies lineage, builds the Merkle tree, and checks
consistency with the on-chain root.

## Sync Algorithm

```
sync(query: &ChiaQuery, config: &NetworkConfig) -> ConsensusResult<NetworkState>
    │
    ├── 1. get_blockchain_state() → peak_height
    │       Detect reorg: if peak < last_synced, rollback to safe checkpoint
    │
    ├── 2. Fetch network coin singleton
    │       get_coin_records_by_puzzle_hash(network_coin_puzzle_hash)
    │       → NetworkCoinState { coin, lineage_proof }
    │
    ├── 3. Fetch checkpoint singleton
    │       get_coin_records_by_puzzle_hash(checkpoint_puzzle_hash)
    │       get_puzzle_and_solution(parent_id) → decurry state
    │       → CheckpointSingletonState { coin, epoch, state_root, merkle_root, count }
    │
    ├── 4. Discover registration coins
    │       get_coin_records_by_puzzle_hash(registration_coin_puzzle_hash, include_spent=false)
    │       For each: verify lineage via get_coin_records_by_parent_ids
    │       → Vec<ValidatorInfo>
    │
    ├── 5. Build sparse Merkle tree from valid registration coins
    │       SparseMerkleTree::from_validators(valid_coins)
    │
    ├── 6. Verify Merkle root consistency
    │       computed_root == checkpoint.validator_merkle_root
    │       → StateMismatch if different
    │
    └── 7. Update cache, return NetworkState
```

### ChiaQuery Methods per Step

| Step | ChiaQuery Method | Parameters |
|------|-----------------|------------|
| 1 | `get_blockchain_state()` | none |
| 2 | `get_coin_records_by_puzzle_hash()` | network coin puzzle hash, unspent only |
| 2 | `get_puzzle_and_solution()` | parent coin ID (for lineage proof) |
| 3 | `get_coin_records_by_puzzle_hash()` | checkpoint puzzle hash, unspent only |
| 3 | `get_puzzle_and_solution()` | parent coin ID (to decurry state) |
| 4 | `get_coin_records_by_puzzle_hash()` | registration coin puzzle hash, unspent only |
| 4 | `get_coin_records_by_parent_ids()` | parent IDs for lineage verification |

### IndexerChain Wrapper

```rust
/// Thin wrapper around ChiaQuery that converts between Bytes32 and hex strings.
pub(crate) struct IndexerChain {
    query: Arc<ChiaQuery>,
}

impl IndexerChain {
    pub async fn get_coin(&self, coin_id: Bytes32) -> ConsensusResult<Option<CoinRecord>> {
        let hex = bytes32_to_hex(&coin_id);
        match self.query.get_coin_record_by_name(&hex).await {
            Ok(record) => Ok(Some(record)),
            Err(ChiaQueryError::...) => Ok(None),  // Not found
            Err(e) => Err(ConsensusError::RpcError(e.to_string())),
        }
    }

    pub async fn get_coins_by_puzzle_hash(
        &self,
        puzzle_hash: Bytes32,
        include_spent: bool,
    ) -> ConsensusResult<Vec<CoinRecord>> {
        let hex = bytes32_to_hex(&puzzle_hash);
        self.query
            .get_coin_records_by_puzzle_hash(&hex, None, None, include_spent)
            .await
            .map_err(|e| ConsensusError::RpcError(e.to_string()))
    }
}
```

### Registration Coin Discovery

```rust
async fn discover_registration_coins(
    chain: &IndexerChain,
    config: &NetworkConfig,
    lineage_checker: &LineageChecker,
) -> ConsensusResult<Vec<ValidatorInfo>> {
    // Query all unspent coins with registration coin puzzle hash prefix
    // Note: registration coins have different puzzle hashes per validator
    // (curried with different pubkeys), so we can't query by a single hash.
    //
    // Strategy: query by parent IDs (all known network coin spend IDs)
    // Each child of a network coin spend with correct amount is a candidate.

    let mut validators = Vec::new();

    for spend_id in lineage_checker.network_coin_spends() {
        let children = chain.get_coins_by_parent_id(*spend_id, false).await?;

        for child in children {
            if child.coin.amount == config.collateral_amount {
                // Extract pubkey from memo (NET-005)
                let spend = chain.get_puzzle_and_solution(child.coin.parent_coin_info).await?;
                if let Some(pubkey) = extract_pubkey_memo(&spend) {
                    // Verify puzzle hash matches
                    let expected_hash = registration_coin_puzzle_hash(
                        pubkey, config.checkpoint_singleton_id(), config
                    );
                    if child.coin.puzzle_hash == expected_hash {
                        validators.push(ValidatorInfo { pubkey, coin: child.coin });
                    }
                }
            }
        }
    }

    Ok(validators)
}
```

## Acceptance Criteria

- [ ] `sync()` returns complete `NetworkState` from live chain
- [ ] Network coin singleton discovered and state extracted
- [ ] Checkpoint singleton state decurried (epoch, roots, count)
- [ ] Registration coins discovered via parent ID queries
- [ ] Lineage verified for each registration coin
- [ ] Merkle tree built from valid registration coins
- [ ] `StateMismatch` returned when computed root != on-chain root
- [ ] Reorg detection via peak height comparison
- [ ] Cache updated after successful sync
