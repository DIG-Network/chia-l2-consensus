# RPC-002 — Puzzle Driver Spend Bundle Construction

> **Authoritative requirement:** [RPC-002](../NORMATIVE.md#RPC-002)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

The 8 `todo!()` stubs in `src/puzzles/*.rs` build spend bundles for the three
on-chain puzzles. Each driver uses `chia-wallet-sdk::SpendContext` for CLVM
allocation and `chia-query::ChiaQuery` to look up current singleton state.

### Singleton Lookup Pattern

All singleton lookups follow the same pattern via `chia-query`:

```rust
async fn fetch_singleton_state(
    query: &ChiaQuery,
    launcher_id: Bytes32,
) -> ConsensusResult<(Coin, /* lineage proof, curried state */)> {
    let puzzle_hash = singleton_puzzle_hash(launcher_id);
    let hex_ph = bytes32_to_hex(&puzzle_hash);

    let records = query
        .get_coin_records_by_puzzle_hash(&hex_ph, None, None, false)
        .await
        .map_err(|e| ConsensusError::RpcError(e.to_string()))?;

    let record = records.into_iter()
        .find(|r| !r.spent)
        .ok_or(ConsensusError::NotDeployed)?;

    // Fetch parent spend to get lineage proof
    let parent_spend = query
        .get_puzzle_and_solution(
            &bytes32_to_hex(&record.coin.parent_coin_info),
            Some(record.confirmed_block_index),
        ).await
        .map_err(|e| ConsensusError::RpcError(e.to_string()))?;

    // Extract lineage proof and curried state from parent spend
    // ...

    Ok((coin, /* lineage, state */))
}
```
