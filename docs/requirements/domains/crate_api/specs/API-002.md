> **Authoritative requirement:** [API-002](../NORMATIVE.md#API-002)

## Summary

`NetworkConfig` must be serializable to/from JSON for persistence across node restarts, and must provide two helper methods for common operations.

## Specification

### Serde Derives

```rust
#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct NetworkConfig { ... }
```

The `Bytes32` fields from `chia_protocol` don't have serde support. Implement custom serialization that encodes `Bytes32` as `0x`-prefixed hex strings.

### verification_key() Method (Spec Line 300-303)

```rust
impl NetworkConfig {
    pub fn verification_key(&self) -> ConsensusResult<VerifyingKey<Bls12_381>> {
        // Deserialize from verification_key_hex field
    }
}
```

### checkpoint_singleton_id() Method (Spec Line 310-312)

```rust
impl NetworkConfig {
    pub fn checkpoint_singleton_id(&self) -> Bytes32 {
        // Derive singleton coin ID from checkpoint_launcher_id
        // Uses SINGLETON_LAUNCHER_PUZZLE_HASH
    }
```

## Acceptance Criteria

- [ ] NetworkConfig derives Serialize + Deserialize
- [ ] JSON round-trip preserves all fields (Bytes32 as hex strings)
- [ ] `verification_key()` returns valid `VerifyingKey<Bls12_381>`
- [ ] `checkpoint_singleton_id()` matches `Launcher::new().coin().coin_id()`

## Gaps Addressed

| Current State | Required State |
|--------------|---------------|
