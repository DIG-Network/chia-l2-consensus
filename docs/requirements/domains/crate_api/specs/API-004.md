## Summary

`ConsensusClient` must provide convenience accessors for the current on-chain state, plus infrastructure methods for cache management and proving key loading.

## Specification

### State Accessors (Spec Lines 2129-2149)

```rust
impl ConsensusClient {
    /// Current epoch from checkpoint singleton (primary health signal).
    pub fn epoch(&self) -> ConsensusResult<u64>;

    /// Current L2 state root.
    pub fn state_root(&self) -> ConsensusResult<Bytes32>;

    /// Current on-chain Merkle root.
    pub fn validator_merkle_root(&self) -> ConsensusResult<Bytes32>;

    /// On-chain count from checkpoint singleton.
    pub fn validator_count(&self) -> ConsensusResult<u64>;

    /// Block height of last sync.
    pub fn synced_at(&self) -> ConsensusResult<u32>;
}
```

These read from the cached `NetworkState` populated by `sync()`. Return `NotDeployed` if `sync()` hasn't been called.

### Infrastructure Methods (Spec Lines 1604-1620)

```rust
impl ConsensusClient {
    /// Set indexer cache file path for fast restarts.
    pub fn set_cache_path(&mut self, path: &str);

    /// Load Groth16 proving key (100-500MB). Only needed on checkpoint submitters.
    pub fn load_proving_key(&mut self, path: &str) -> ConsensusResult<()>;
}
```

### Internal State

The client needs an `Option<NetworkState>` field to store synced state:

```rust
pub struct ConsensusClient {
    config: NetworkConfig,
    indexer: IndexerState,
    state: Option<NetworkState>,           // Populated by sync()
    proving_key: Option<ProvingKey>,       // Populated by load_proving_key()
    cache_path: Option<String>,            // Set by set_cache_path()
}
```

## Acceptance Criteria

- [ ] `epoch()` returns current epoch from synced state
- [ ] `state_root()` returns current state root
- [ ] `validator_merkle_root()` returns current Merkle root
- [ ] `validator_count()` returns current count
- [ ] `synced_at()` returns block height of last sync
- [ ] All accessors return `NotDeployed` before first `sync()`
- [ ] `set_cache_path()` stores path for indexer cache
- [ ] `load_proving_key()` loads and deserializes PK
