# API-001 — Public API Surface

> **Authoritative requirement:** [API-001](../NORMATIVE.md#API-001)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [spec-consensus-crate.md](../../../../resources/spec-consensus-crate.md) Lines 24-55, 2154-2181

## Summary

The crate's public API must be the minimal set of types and functions that L2 consumers need. Internal implementation details (wire format helpers, circuit types, aggregation functions) are currently exported directly from `lib.rs` but should be accessible only via a `pub mod testing` module for VV tests.

## Specification

### Public API (for L2 consumers)

Per spec Lines 24-55, L2 systems use only:
- `ConsensusClient` — main entry point
- `NetworkConfig` — deployment parameters
- `DeploymentArtifacts`, `VkJson` — deployment artifact serialization
- `ConsensusError`, `ConsensusResult` — error types
- `CheckpointSingletonState`, `NetworkCoinState`, `NetworkState` — on-chain state
- `ValidatorSet`, `ValidatorInfo` — validator set types
- `SpendBundle`, `Bytes32` — Chia protocol types (re-exported)

### Testing API (for VV tests)

All current exports from `prover::*`, `validator::*`, `puzzles::*`, `merkle::*`, `indexer::*` move to:

```rust
/// Internal types and functions re-exported for integration testing.
/// Not part of the stable public API for L2 consumers.
pub mod testing {
    pub use crate::prover::{...};
    pub use crate::validator::{...};
    pub use crate::puzzles::{...};
    pub use crate::merkle::{...};
    pub use crate::indexer::{...};
}
```

### Migration for VV Tests

All existing `use chia_l2_consensus::{internal_fn}` imports in `tests/vv_req_*.rs` become `use chia_l2_consensus::testing::{internal_fn}`.

## Acceptance Criteria

- [ ] `lib.rs` exports only spec-defined public types
- [ ] `pub mod testing` re-exports all internal functions VV tests need
- [ ] All 66+ existing VV tests compile and pass after migration
- [ ] L2 consumer code only needs `ConsensusClient` + returned types

## Gaps Addressed

| Current State | Required State |
|--------------|---------------|
| ~50 functions exported directly from lib.rs | Minimal public API + `testing` module |
| `pub mod indexer`, `pub mod merkle` | `pub(crate) mod` with selected re-exports |
| Internal prover/validator/puzzle exports at top level | Behind `testing` module |
