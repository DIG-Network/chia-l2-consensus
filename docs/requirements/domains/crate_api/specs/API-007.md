## Summary

API-001 (public API surface) and API-006 (module visibility) rely on
compilation-verified status — if the exports were wrong, other test files would
fail to compile. While this provides implicit verification, dedicated test files
make the coverage explicit and auditable.

## Specification

### vv_req_api_001.rs (public API surface)
- Import `chia_l2_consensus` as an external crate
- Verify each public type listed in spec can be constructed/referenced
- Verify `testing` module is accessible
- Verify internal types (e.g., `SparseMerkleTree`) are NOT directly accessible
  from external crate position (compile-fail test pattern or cfg(test) guard)

### vv_req_api_006.rs (module visibility)
- Verify `testing::` re-exports match expected list
- Verify key internal types are accessible via testing module
- Verify module visibility constraints are documented in lib.rs
