# API-003 — State Types

> **Authoritative requirement:** [API-003](../NORMATIVE.md#API-003)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [spec-consensus-crate.md](../../../../resources/spec-consensus-crate.md) Lines 319-409

## Summary

The on-chain state types must match the spec: `NetworkState` (aggregate), `ValidatorInfo` (single validator), and updated `ValidatorSet` with helpers. Singleton state types need `lineage_proof` fields for building spend bundles.

## Specification

### NetworkState (Spec Lines 363-377) — MISSING

```rust
pub struct NetworkState {
    pub network_coin:  NetworkCoinState,
    pub checkpoint:    CheckpointSingletonState,
    pub validators:    ValidatorSet,
    pub merkle_tree:   SparseMerkleTree,
    pub synced_at:     u32,
}
```

### ValidatorInfo (Spec Lines 389-396) — MISSING

```rust
pub struct ValidatorInfo {
    pub pubkey:            [u8; 48],   // BLS12-381 G1, 48 bytes compressed
    pub registration_coin: Coin,       // Unspent registration coin
}
```

### ValidatorSet Update (Spec Lines 383-408) — WRONG INNER TYPE

Current uses `Vec<Validator>` with `slot: u32` and `registration_coin_id: Bytes32`.
Spec requires `Vec<ValidatorInfo>` with `pubkey` and `registration_coin`.

### ValidatorSet Helper Methods (Spec Lines 399-408) — MISSING

```rust
impl ValidatorSet {
    pub fn count(&self) -> u64;
    pub fn contains(&self, pubkey: &[u8; 48]) -> bool;
    pub fn pubkeys(&self) -> Vec<[u8; 48]>;
}
```

### NetworkCoinState.lineage_proof (Spec Line 333) — MISSING FIELD

```rust
pub struct NetworkCoinState {
    pub coin: Coin,
    pub lineage_proof: Proof,  // from chia_puzzles
}
```

### CheckpointSingletonState.lineage_proof (Spec Line 347) — MISSING FIELD

```rust
pub struct CheckpointSingletonState {
    pub coin: Coin,
    pub lineage_proof: Proof,  // from chia_puzzles
    pub epoch: u64,
    pub validator_count: u64,
    pub validator_merkle_root: Bytes32,
    pub state_root: Bytes32,

```

## Acceptance Criteria

- [ ] `NetworkState` struct exists with all 5 fields
- [ ] `ValidatorInfo` struct exists with pubkey + registration_coin
- [ ] `ValidatorSet` uses `Vec<ValidatorInfo>` (not `Vec<Validator>`)
- [ ] `ValidatorSet::count()`, `contains()`, `pubkeys()` work correctly
- [ ] `NetworkCoinState` has `lineage_proof` field
- [ ] `CheckpointSingletonState` has `lineage_proof` field
- [ ] Old `Validator` struct removed or deprecated

## Gaps Addressed

| Current State | Required State |
|--------------|---------------|
| No `NetworkState` | Add aggregate state struct |
| No `ValidatorInfo` | Add per-validator struct |
| `ValidatorSet { validators: Vec<Validator> }` | `ValidatorSet { validators: Vec<ValidatorInfo> }` |
| No helper methods on `ValidatorSet` | Add `count()`, `contains()`, `pubkeys()` |
| `NetworkCoinState` has only `coin` | Add `lineage_proof` |
| `CheckpointSingletonState` missing `lineage_proof` | Add field |
