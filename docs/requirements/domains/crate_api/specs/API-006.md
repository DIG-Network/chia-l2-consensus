## Summary

Internal modules must use `pub(crate)` visibility. Selected types are re-exported through `lib.rs`. VV test access to internals is provided via a `pub mod testing` module.

## Specification

### Module Visibility

```rust
// src/lib.rs
mod client;
mod config;
mod error;
mod state;

pub(crate) mod indexer;
pub(crate) mod merkle;
pub(crate) mod prover;
pub(crate) mod puzzles;
pub(crate) mod validator;
```

### VV Test Migration

Every `tests/vv_req_*.rs` file changes:
```rust
// Before:
use chia_l2_consensus::{compute_checkpoint_message, is_majority, ...};

// After:
use chia_l2_consensus::testing::{compute_checkpoint_message, is_majority, ...};
```

Public API types (`ConsensusClient`, `NetworkConfig`, etc.) remain at `chia_l2_consensus::`.

## Acceptance Criteria

- [ ] `indexer`, `merkle`, `prover`, `puzzles`, `validator` are all `pub(crate)`
- [ ] `pub mod testing` re-exports everything VV tests need
- [ ] All 66+ existing VV tests compile and pass without modification to logic
- [ ] Only import paths change in test files (add `testing::` prefix)
- [ ] L2 consumer code can import `ConsensusClient` and `NetworkConfig` without `testing`

## Gaps Addressed

| Current State | Required State |
|--------------|---------------|
| `pub mod indexer`, `pub mod merkle` | `pub(crate) mod` |
| ~50 direct exports in lib.rs | Minimal public + `testing` module |
| No separation of public vs test API | Clear boundary |
