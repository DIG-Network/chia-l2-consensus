## Summary

The validator set is stored in a sparse Merkle tree (SMT) with a fixed depth of `TREE_DEPTH` (typically 32). This structure supports efficient membership and non-membership proofs with O(depth) complexity while accommodating dynamic validator sets.

## Specification

### Tree Properties

| Property | Value | Description |
|----------|-------|-------------|
| Type | Sparse Merkle Tree | Fixed-depth, deterministic placement |
| Depth | TREE_DEPTH (32) | Set at circuit compile time |
| Capacity | 2^TREE_DEPTH | ~4.3 billion slots at depth 32 |
| Leaf size | 32 bytes | SHA-256 hash |
| Node size | 32 bytes | SHA-256 hash of children |

### Why Sparse Merkle Tree

**vs. Standard Merkle Tree:**
- Standard trees grow with number of entries
- SMT has fixed depth regardless of validator count
- SMT supports non-membership proofs natively

**vs. Merkle Patricia Trie:**
- MPT has variable-depth paths
- SMT has uniform proof size (always TREE_DEPTH)
- SMT is simpler to implement in ZK circuits

### Tree Structure

```
Level 0 (root):     [root_hash]
                    /          \
Level 1:        [h01]          [h02]
               /    \          /    \
Level 2:    [h11]  [h12]    [h21]  [h22]
            ...    ...      ...    ...
Level 31:   ...    ...      ...    ...
Level 32:   [leaf] [leaf]  [leaf] [leaf]  (2^32 leaves)
```

### Node Hash Computation

```
parent = sha256(left_child || right_child)
```

Always concatenate left child first, then right child.

### Leaf Level

At level TREE_DEPTH (the bottom):
- Each position corresponds to a validator slot
- Active slot: `sha256(pubkey)`
- Empty slot: `EMPTY_LEAF_HASH`

### Depth Consistency

All operations must respect TREE_DEPTH:
- Proof generation: exactly TREE_DEPTH siblings
- Proof verification: exactly TREE_DEPTH hash operations
- Root computation: exactly TREE_DEPTH levels
- Circuit: compiled for TREE_DEPTH

## Acceptance Criteria

- [ ] Tree depth is configurable constant TREE_DEPTH
- [ ] Default TREE_DEPTH = 32
- [ ] All proofs have exactly TREE_DEPTH siblings
- [ ] Root computation traverses exactly TREE_DEPTH levels
- [ ] Tree supports 2^TREE_DEPTH slots
- [ ] Same TREE_DEPTH used in circuit, Rust, and Chialisp

## Implementation Notes

- **Codebase:** `src/sparse_merkle_tree.rs`
- **Constant:** `const TREE_DEPTH: usize = 32;`

### Data Structure

```rust
pub struct SparseMerkleTree {
    depth: usize,
    leaves: HashMap<u64, [u8; 32]>,  // slot → leaf hash
    empty_hashes: Vec<[u8; 32]>,      // precomputed empty subtree hashes
}
```

## Verification

1. Create tree with depth 32
2. Insert validator at slot 0
3. Verify proof has 32 siblings
4. Verify root computation uses 32 levels
