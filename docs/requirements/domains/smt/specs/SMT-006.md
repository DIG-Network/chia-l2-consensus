# SMT-006 — Empty Tree Optimization

> **Authoritative requirement:** [SMT-006](../NORMATIVE.md#SMT-006)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **CHIP reference:** [chip-groth16-l2-consensus.md](../../../../resources/chip-groth16-l2-consensus.md) — Sparse Merkle Tree

## Summary

Computing hashes for empty subtrees repeatedly is wasteful. Precomputed empty hashes at each tree level enable O(n × depth) root computation instead of O(2^depth), making the sparse tree practical.

## Specification

### Empty Hash Array

Precompute empty subtree hashes for each level:

```rust
empty_hashes: [
    EMPTY_LEAF_HASH,           // Level 32 (leaf)
    sha256(empty[32] || empty[32]),  // Level 31
    sha256(empty[31] || empty[31]),  // Level 30
    ...
    sha256(empty[1] || empty[1]),    // Level 0 (root of empty tree)
]
```

### Computation

```rust
fn precompute_empty_hashes(depth: usize) -> Vec<[u8; 32]> {
    let mut hashes = Vec::with_capacity(depth + 1);

    // Level depth (leaves)
    hashes.push(EMPTY_LEAF_HASH);

    // Levels depth-1 down to 0
    for _ in 0..depth {
        let prev = hashes.last().unwrap();
        let next = sha256(&[prev, prev].concat());
        hashes.push(next);
    }

    hashes
}
```

### Usage in Root Computation

When computing root with sparse entries:

```rust
fn compute_root(&self) -> [u8; 32] {
    self.compute_node(0, 0)  // Start at root position
}

fn compute_node(&self, level: usize, index: u64) -> [u8; 32] {
    if level == self.depth {
        // Leaf level
        self.leaves.get(&index)
            .copied()
            .unwrap_or(EMPTY_LEAF_HASH)
    } else {
        // Check if entire subtree is empty
        if self.is_subtree_empty(level, index) {
            return self.empty_hashes[self.depth - level];
        }

        // Recursively compute children
        let left = self.compute_node(level + 1, index * 2);
        let right = self.compute_node(level + 1, index * 2 + 1);
        sha256(&[left, right].concat())
    }
}
```

### Empty Root Constant

For depth 32, the empty tree root is a known constant:

```rust
pub const EMPTY_TREE_ROOT_32: [u8; 32] = /* precomputed value */;
```

This is the initial `validator_merkle_root` at deployment (before any validators).

### Optimization Impact

| Approach | Complexity | For 10,000 validators |
|----------|------------|----------------------|
| Naive | O(2^depth) | 4 billion hashes |
| With empty optimization | O(n × depth) | 320,000 hashes |

### Memory Efficiency

Instead of storing 2^32 leaves:
- Store only active leaves: O(n) entries
- Store empty hashes: O(depth) entries
- Lazy computation on demand

### Chialisp Consideration

On-chain (Chialisp), empty hashes are constants:
- EMPTY_LEAF_HASH curried into checkpoint singleton
- No array needed — only used for non-membership proofs
- Verifier checks leaf == EMPTY_LEAF_HASH for non-membership

## Acceptance Criteria

- [ ] Empty hashes precomputed at initialization
- [ ] Empty subtrees return precomputed hash, not recompute
- [ ] Root computation is O(n × depth) not O(2^depth)
- [ ] Empty tree root is known constant
- [ ] Initial deployment uses empty tree root
- [ ] Memory usage is O(n + depth) not O(2^depth)

### Computing Empty Tree Root

```rust
pub fn empty_tree_root(depth: usize) -> [u8; 32] {
    let hashes = precompute_empty_hashes(depth);
    hashes[depth]  // Root is last element
}

// For depth 32
lazy_static! {
    pub static ref EMPTY_TREE_ROOT: [u8; 32] = empty_tree_root(32);
}
```

## Verification

1. Compute empty_hashes[0] = sha256(48 zeros)
2. Compute empty_hashes[1] = sha256(empty[0] || empty[0])
3. Continue to level 32
4. Verify final value matches EMPTY_TREE_ROOT constant
5. Build empty tree; verify root() == EMPTY_TREE_ROOT
6. Benchmark: verify O(n × depth) scaling
