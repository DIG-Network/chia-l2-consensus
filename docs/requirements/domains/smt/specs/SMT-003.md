# SMT-003 — Leaf Values

> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

Leaf values distinguish between active validators and empty slots. Active slots contain the hash of the validator's pubkey; empty slots contain a fixed constant hash. This distinction enables both membership and non-membership proofs.

## Specification

### Active Leaf

For a registered validator at their computed slot:
```
active_leaf = sha256(pubkey)
```

Where `pubkey` is the 48-byte compressed BLS12-381 G1 point.

### Empty Leaf

For slots without a validator:
```
EMPTY_LEAF_HASH = sha256(0x00 × 48)
                = sha256([0,0,0,...,0])  // 48 zero bytes
```

This is a known constant, precomputed once:
```
EMPTY_LEAF_HASH = 0x7d4e3eec80a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8091a2b3c4d5e6f7081
```

(Actual value computed from 48 zeros)

### Why sha256(pubkey) Not Just pubkey

1. **Uniform size:** All leaves are 32 bytes (pubkey is 48 bytes)
2. **Collision resistance:** Even if pubkey format changes
3. **Circuit efficiency:** 32-byte inputs to parent hash
4. **Hiding:** Slightly obscures pubkey in tree structure

### Why sha256(zeros) Not Just zeros

1. **Non-trivial:** Zero bytes aren't "nothing" — they're explicitly empty
2. **Distinguishable:** EMPTY_LEAF_HASH ≠ any realistic sha256(pubkey)
3. **Consistent:** Same pattern as active leaves (hash of content)

### Leaf Operations

**Insert validator:**
```rust
tree.set_leaf(slot, sha256(&pubkey));
```

**Remove validator:**
```rust
tree.set_leaf(slot, EMPTY_LEAF_HASH);
```

**Check membership:**
```rust
let is_member = tree.get_leaf(slot) == sha256(&pubkey);
let is_empty = tree.get_leaf(slot) == EMPTY_LEAF_HASH;
```

### Proof Types

| Query | Expected Leaf | Proves |
|-------|--------------|--------|
| Membership | sha256(pubkey) | Validator IS registered |
| Non-membership | EMPTY_LEAF_HASH | Validator is NOT registered |

### Circuit Usage

In the Groth16 circuit:
```rust
// Membership check
let leaf = sha256_gadget(pubkey);
verify_merkle_path(leaf, proof, index, root);

// On-chain non-membership check (checkpoint singleton)
let leaf = EMPTY_LEAF_HASH;  // Constant
verify_merkle_path(leaf, proof, index, root);
```

## Acceptance Criteria

- [ ] Active leaf = sha256(pubkey)
- [ ] Empty leaf = sha256(48 zero bytes)
- [ ] EMPTY_LEAF_HASH is a known constant
- [ ] Membership proof uses sha256(pubkey) as leaf
- [ ] Non-membership proof uses EMPTY_LEAF_HASH
- [ ] Same EMPTY_LEAF_HASH in Rust and Chialisp

## Implementation Notes

- **Constant:** `const EMPTY_LEAF_HASH: [u8; 32] = sha256([0u8; 48]);`
- **Precomputed:** Calculate once at compile time or initialization

### Computing EMPTY_LEAF_HASH

```rust
lazy_static! {
    pub static ref EMPTY_LEAF_HASH: [u8; 32] = {
        let zero_pubkey = [0u8; 48];
        sha256(&zero_pubkey)
    };
}
```

## Verification

1. Compute sha256(48 zeros) manually
2. Verify matches EMPTY_LEAF_HASH constant
3. Insert pubkey; verify leaf = sha256(pubkey)
4. Remove pubkey; verify leaf = EMPTY_LEAF_HASH
5. Cross-check constants between Rust and Chialisp
