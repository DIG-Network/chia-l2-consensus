## Summary

Merkle proofs consist of exactly TREE_DEPTH sibling hashes. The sibling ordering convention determines how to reconstruct the path from leaf to root. This format must be identical between off-chain proof generation and on-chain verification.

## Specification

### Proof Structure

```rust
pub struct MerkleProof {
    pub leaf_index: u64,           // Slot position
    pub siblings: [[u8; 32]; 32],  // Exactly TREE_DEPTH hashes
}
```

### Sibling Array

- Length: exactly TREE_DEPTH (32 for depth-32 tree)
- Order: bottom-up (siblings[0] is at leaf level, siblings[31] near root)
- Each entry: 32-byte SHA-256 hash

### Left/Right Convention

At each level, determine if current node is left or right child:

```rust
fn is_left_child(index: u64, level: usize) -> bool {
    (index >> level) & 1 == 0
}
```

- If left child (bit = 0): sibling is on right
- If right child (bit = 1): sibling is on left

### Hash Computation

**Critical convention: left child ALWAYS first**

```rust
fn compute_parent(current: &[u8; 32], sibling: &[u8; 32], is_left: bool) -> [u8; 32] {
    if is_left {
        sha256(&[current, sibling].concat())  // current || sibling
    } else {
        sha256(&[sibling, current].concat())  // sibling || current
    }
}
```

### Proof Verification Algorithm

```rust
fn verify_proof(
    leaf: [u8; 32],
    proof: &MerkleProof,
    expected_root: [u8; 32],
    tree_depth: usize,
) -> bool {
    let mut current = leaf;

    for level in 0..tree_depth {
        let sibling = proof.siblings[level];
        let is_left = is_left_child(proof.leaf_index, level);
        current = compute_parent(&current, &sibling, is_left);
    }

    current == expected_root
}
```

### Example (Depth 4)

```
Tree:
           [R]           Level 0 (root)
          /   \
        [A]   [B]        Level 1
        / \   / \
      [C][D][E][F]       Level 2
      /\ /\ /\ /\
     ... ... ... ...     Level 3 (leaves)

Proof for leaf at index 5 (binary: 0101):
- Level 0: bit 1 = right child → sibling[0] = left neighbor
- Level 1: bit 0 = left child → sibling[1] = right neighbor
- Level 2: bit 1 = right child → sibling[2] = left neighbor
- Level 3: bit 0 = left child → sibling[3] = right neighbor

siblings = [neighbor_at_level_0, neighbor_at_level_1, neighbor_at_level_2, neighbor_at_level_3]
```

### Chialisp Equivalent

```clojure
(defun verify_path (current siblings index depth)
    (if (= depth 0)
        current
        (verify_path
            (if (logand index 1)
                (sha256 (first siblings) current)   ; Right child: sibling || current
                (sha256 current (first siblings)))  ; Left child: current || sibling
            (rest siblings)
            (lsh index -1)
            (- depth 1))))
```

## Acceptance Criteria

- [ ] Proof contains exactly TREE_DEPTH siblings
- [ ] Siblings ordered bottom-up (leaf level first)
- [ ] Left child determination: (index >> level) & 1 == 0
- [ ] Hash always: sha256(left || right)
- [ ] Rust and Chialisp use identical convention
- [ ] Valid proof reconstructs correct root

## Implementation Notes

- **Codebase:** `src/sparse_merkle_tree.rs`
- **Common bug:** Swapping left/right or reversing sibling order

### Test Vector

For consistent testing across implementations:

```rust
// Known test case
let pubkey = [0x01; 48];  // All 1s
let leaf = sha256(&pubkey);
let index = compute_slot(&pubkey, 32);
// ... generate proof and verify against known root
```

## Verification

1. Generate proof for known leaf
2. Verify reconstruction produces expected root
3. Corrupt one sibling → verification fails
4. Swap two siblings → verification fails
5. Cross-check: Rust proof verifies in Chialisp
