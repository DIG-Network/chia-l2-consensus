## Summary

Each validator's position in the tree is determined by hashing their public key, ensuring deterministic and collision-resistant slot assignment. This allows any party to independently compute where a validator belongs in the tree.

## Specification

### Slot Computation

```
slot = first_8_bytes_as_u64_be(sha256(pubkey)) mod 2^TREE_DEPTH
```

Step by step:
1. Hash the pubkey: `h = sha256(pubkey)` (32 bytes)
2. Take first 8 bytes: `h[0..8]`
3. Interpret as big-endian u64: `n = u64::from_be_bytes(h[0..8])`
4. Modulo tree capacity: `slot = n % (1 << TREE_DEPTH)`

### Example

```rust
let pubkey = hex::decode("b5a2...48 bytes")?;
let hash = sha256(&pubkey);
// hash = 0x7d4e3eec80a1b2c3...
let n = u64::from_be_bytes(hash[0..8].try_into()?);
// n = 0x7d4e3eec80a1b2c3 = 9,026,891,547,234,567,875
let slot = n % (1 << 32);
// slot = 0x80a1b2c3 = 2,158,883,523
```

### Why This Design

**Deterministic:** Given a pubkey, any party computes the same slot.

**Collision-resistant:** SHA-256 provides uniform distribution, making collisions unlikely for realistic validator counts.

**Simple:** Single hash operation, easy to implement in Rust and Chialisp.

### Collision Handling

Two pubkeys could theoretically hash to the same slot:
- Probability: ~n²/2^64 for n validators (birthday bound on first 8 bytes)
- With 10,000 validators: ~0.5 × 10^8 / 2^64 ≈ negligible

If collision occurs:
- Second registration MUST be rejected at L2 level
- Indexer detects and reports `SlotCollision` error
- First validator retains the slot

### Implementation

```rust
pub fn compute_slot(pubkey: &[u8; 48], tree_depth: usize) -> u64 {
    let hash = sha256(pubkey);
    let n = u64::from_be_bytes(hash[0..8].try_into().unwrap());
    n % (1u64 << tree_depth)
}
```

### Chialisp Equivalent

```clojure
(defun compute_slot (pubkey tree_depth)
    (logand
        (/ (sha256 pubkey) (exp 2 (- 256 64)))  ; First 8 bytes as integer
        (- (exp 2 tree_depth) 1)))               ; Mask for tree_depth bits
```

## Acceptance Criteria

- [ ] Slot computed from sha256(pubkey)
- [ ] First 8 bytes interpreted as big-endian u64
- [ ] Result reduced mod 2^TREE_DEPTH
- [ ] Same pubkey → same slot every time
- [ ] Different pubkeys → different slots (with high probability)
- [ ] Rust and Chialisp compute identical slots

## Implementation Notes

- **Codebase:** `src/sparse_merkle_tree.rs`
- **Collision detection:** `src/indexer.rs`

## Verification

1. Compute slot for known pubkey manually
2. Verify implementation produces same result
3. Compute slots for 10,000 random pubkeys
4. Verify no collisions (statistical test)
5. Cross-check Rust slot == Chialisp slot
