# VAL-003 — Signing Protocol

> **Authoritative requirement:** [VAL-003](../NORMATIVE.md#VAL-003)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **CHIP reference:** [chip-groth16-l2-consensus.md](../../../../resources/chip-groth16-l2-consensus.md) — Circuit design choice

## Summary

When signing checkpoints, validators sign the message `checkpoint_message || genesis_challenge || checkpoint_singleton_coin_id` using their BLS secret key. Signatures are only provided for valid checkpoint messages.

## Specification

### Signing Message Format

```
signing_message = checkpoint_message + genesis_challenge + coin_id
```

Where:
- `checkpoint_message`: 32 bytes from WIRE-001
- `genesis_challenge`: 32 bytes from network config
- `coin_id`: 32 bytes, current checkpoint singleton coin ID

Total: 96 bytes.

### Signing Conditions

A validator SHOULD only sign if:
1. They have independently verified the checkpoint message
2. The new state root is valid per L2 rules
3. The new validator merkle root matches their local computation
4. The epoch is exactly current_epoch + 1

### Signature Output

- BLS signature over signing_message
- G2 point, 96 bytes compressed
- Shared with checkpoint coordinator for aggregation

### Why This Message

| Component | Purpose |
|-----------|---------|
| checkpoint_message | Commits to all new state |
| genesis_challenge | Network binding (prevents cross-network replay) |
| coin_id | Checkpoint binding (changes each spend) |

## Acceptance Criteria

- [ ] Validator signs correct 96-byte message
- [ ] Signature uses BLS12-381 G2
- [ ] Only valid checkpoint messages are signed
- [ ] Independent verification before signing
- [ ] Signature aggregates correctly with others

## Implementation Notes

- **Tool:** `l2-validator sign`
- **Library:** `blst` min_pk
- **Output:** 96-byte G2 signature

### Signing Command

```bash
./target/release/l2-validator sign \
  --config validator_config.toml \
  --message <checkpoint_message_hex>

# Output:
# Signing message: <hex>
# Signature: <96-byte hex G2 point>
# Public key: <48-byte hex G1 point>
```

### Signing Code

```rust
pub fn sign_checkpoint(
    secret_key: &SecretKey,
    checkpoint_message: [u8; 32],
    genesis_challenge: [u8; 32],
    checkpoint_coin_id: [u8; 32],
) -> Signature {
    let mut msg = Vec::with_capacity(96);
    msg.extend_from_slice(&checkpoint_message);
    msg.extend_from_slice(&genesis_challenge);
    msg.extend_from_slice(&checkpoint_coin_id);

    secret_key.sign(&msg)
}
```

## Verification

1. Generate valid checkpoint message
2. Sign with validator key
3. Verify signature with public key
4. Test aggregation with multiple signatures
5. Verify aggregated signature verifies on-chain
