# VAL-001 — Key Generation

> **Authoritative requirement:** [VAL-001](../NORMATIVE.md#VAL-001)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **CHIP reference:** [chip-groth16-l2-consensus.md](../../../../resources/chip-groth16-l2-consensus.md) — De-Registration and Collateral Recovery

## Summary

Validators must generate a BLS12-381 keypair and securely back up the secret key before registration. Loss of the secret key results in permanent loss of signing capability and complicates collateral recovery.

## Specification

### Key Type

- Curve: BLS12-381
- Public key: G1 point, 48 bytes compressed
- Private key: Scalar in Fr, 32 bytes

### Key Purpose

| Purpose | How Key Is Used |
|---------|-----------------|
| Identity | Public key determines slot in Merkle tree |
| Signing | Private key signs checkpoint messages |
| Registration | Signature proves key ownership |
| Collateral | Private key needed to authorize recovery destination |

### Key Loss Consequences

If private key is lost:
- Cannot sign checkpoint messages
- Cannot authorize collateral recovery
- Must wait for network to force-exit
- Collateral recovery destination cannot be changed

### Backup Requirements

- Store backup offline
- At least two separate secure locations
- Test recovery before depositing collateral
- Never store on networked computer in plaintext

## Acceptance Criteria

- [ ] Validator can generate BLS12-381 keypair
- [ ] Public key is 48-byte compressed G1
- [ ] Private key is securely stored
- [ ] Backup procedure is documented
- [ ] Recovery can be tested before registration

## Implementation Notes

- **Tool:** `l2-validator keygen`
- **Output:** `validator_key.json`
- **Library:** `blst` or `bls12_381`

### Key Generation

```bash
./target/release/l2-validator keygen \
  --output validator_key.json

# Output:
# Public key:  <48-byte hex>
# Private key: stored in validator_key.json
```

### Key File Format

```json
{
  "version": 1,
  "public_key": "0xb7f1d3a7...",
  "encrypted_secret_key": "...",
  "key_derivation": "argon2id"
}
```

### Verification

```bash
./target/release/l2-validator keyinfo --key validator_key.json
# Expected: Public key: <48-byte hex>, Key type: BLS12-381
```

## Verification

1. Generate keypair
2. Export public key and verify 48 bytes
3. Sign test message with private key
4. Verify signature with public key
5. Backup and restore key file
6. Verify restored key works
