# VAL-004 — Voluntary Exit

> **Authoritative requirement:** [VAL-004](../NORMATIVE.md#VAL-004)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **CHIP reference:** [chip-groth16-l2-consensus.md](../../../../resources/chip-groth16-l2-consensus.md) — De-Registration and Collateral Recovery

## Summary

To exit voluntarily, a validator signals intent at the L2 level, waits for a checkpoint that excludes them, then recovers collateral using a membership query and registration coin spend in the same bundle.

## Specification

### Exit Process

1. **Signal intent**: Contact L2 coordinator to request exclusion
2. **Wait for exclusion**: Next checkpoint removes pubkey from Merkle root
3. **Verify exclusion**: Check status shows INACTIVE
4. **Recover collateral**: Submit two-spend bundle

### Collateral Recovery Bundle

The recovery requires two atomic spends:

**Spend 1: Membership Query**
- Spends checkpoint singleton via membership query path
- Provides non-membership Merkle proof
- Emits non-membership announcement
- Singleton recreates unchanged

**Spend 2: Registration Coin**
- Asserts non-membership announcement from Spend 1
- Returns collateral to specified destination
- Registration coin is consumed

### Timing

| Phase | Duration |
|-------|----------|
| Signal intent | Immediate |
| Wait for exclusion | Until next checkpoint |
| Recovery spend | Any time after exclusion confirmed |

There is no timing pressure on the recovery spend once exclusion is confirmed.

## Acceptance Criteria

- [ ] Validator can signal exit intent
- [ ] Checkpoint excludes validator from Merkle root
- [ ] Non-membership proof is valid
- [ ] Recovery bundle is atomic
- [ ] Collateral returns to specified destination

## Implementation Notes

- **Tool:** `l2-validator exit`
- **API:** `ConsensusClient.recover_collateral()`
- **Cost:** ~7.4M CLVM units

### Exit Command

```bash
./target/release/l2-validator exit \
  --config validator_config.toml \
  --destination <your_wallet_puzzle_hash>

# Output:
# Building collateral recovery spend bundle...
# Membership query spend: OK
# Registration coin spend: OK
# Submitting bundle...
# Bundle ID: <hex>
# Collateral recovery submitted
```

### Exit Flow

```rust
pub async fn recover_collateral(
    &self,
    destination: PuzzleHash,
) -> Result<SpendBundle, ConsensusError> {
    // 1. Verify we are excluded
    let proof = self.merkle_tree.non_membership_proof(&self.pubkey)?;

    // 2. Build membership query spend
    let query_spend = self.build_membership_query_spend(proof)?;

    // 3. Build registration coin spend
    let recovery_spend = self.build_registration_coin_spend(destination)?;

    // 4. Combine atomically
    Ok(SpendBundle::aggregate(vec![query_spend, recovery_spend]))
}
```

## Verification

1. Register as validator
2. Signal exit at L2 level
3. Wait for checkpoint that excludes
4. Submit recovery bundle
5. Verify collateral arrives at destination
6. Verify registration coin is consumed
